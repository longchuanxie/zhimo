# 前端路由与状态管理设计 v1.0

## 1. 路由原则

MVP 是桌面客户端，但前端仍建议使用内部路由管理页面状态。

路由必须中文 UI、英文内部路径。

---

## 2. 路由表

| 页面 | 路径 | 组件 | 数据依赖 |
|---|---|---|---|
| 本地项目库 | `/projects` | ProjectListPage | ProjectService.listProjects |
| 创建项目 | `/projects/new` | CreateProjectPage | ProjectService.createProject |
| 项目首页 | `/projects/:projectId` | ProjectOverviewPage | ProjectService.getProjectOverview |
| 文档列表 | `/projects/:projectId/documents` | DocumentListPage | DocumentService.listDocuments |
| 文档编辑 | `/projects/:projectId/documents/:documentId` | DocumentEditorPage | DocumentService.getDocument |
| 资料库 | `/projects/:projectId/sources` | SourceListPage | SourceService.listSources |
| 资料详情 | `/projects/:projectId/sources/:sourceId` | SourceDetailPage | SourceService.getSource |
| 卡片库 | `/projects/:projectId/cards` | CardListPage | CardService.listCards |
| 大纲 | `/projects/:projectId/outline` | OutlinePage | OutlineService.getOutline |
| 知识库 | `/projects/:projectId/knowledge` | KnowledgePage | KnowledgeService.listKnowledge |
| 助手对话 | `/projects/:projectId/agent` | AgentThreadListPage | AgentService.listThreads |
| 模型设置 | `/settings/models` | ModelSettingsPage | ModelService.listProviders |
| 导出 | `/projects/:projectId/export` | ExportPage | ExportService.listExportTasks |

---

## 3. 页面状态模型

每个页面必须处理：

```text
loading
empty
ready
error
```

示例：

```ts
type PageState<T> =
  | { status: 'loading' }
  | { status: 'empty' }
  | { status: 'ready'; data: T }
  | { status: 'error'; error: AppError }
```

---

## 4. 状态管理选型

建议：

```text
TanStack Query：服务端/本地服务数据缓存
Zustand：客户端 UI 状态
React Hook Form：表单状态
```

```ts

['document', projectId, documentId]
['sources', projectId]

['context-preview', projectId, boundObjectType, boundObjectId]
['model-providers']

```

## 6. Zustand Store 拆分

```text
useAppStore
useProjectStore
useEditorStore
useAgentPanelStore
useTaskStore
useSettingsStore
```

### useAgentPanelStore

```ts
type AgentPanelState = {
  open: boolean
  threadId?: string
  boundObjectType?: string
  boundObjectId?: string
  selectedText?: string
}
```

### useEditorStore

```ts
type EditorState = {
  activeDocumentId?: string
  selectedText?: string
  selectionRange?: {
    from: number
    to: number
  }
  autosaveStatus: 'idle' | 'saving' | 'saved' | 'failed'
}
```

## 7. 缓存失效规则

| 操作 | 失效 Query |
|---|---|
| 创建项目 | `['projects']` |
| 新建文档 | `['documents', projectId]`, `['project-overview', projectId]` |
| 自动保存文档 | `['document', projectId, documentId]`, `['documents', projectId]` |
| 导入资料 | `['sources', projectId]`, `['tasks', projectId]` |
| 生成卡片 | `['cards', projectId]`, `['project-overview', projectId]` |
| 保存为知识 | `['knowledge', projectId]` |
| 发送助手消息 | `['agent-messages', threadId]`, `['agent-threads', projectId]` |
| 创建导出任务 | `['export-tasks', projectId]`, `['tasks', projectId]` |

---

## 8. 可维护性要求

禁止页面自行维护复杂业务缓存。

复杂状态必须拆到：

```text
hooks
stores
services
```

页面组件只负责：

```text
取数
分支渲染
组合组件
绑定动作
```
