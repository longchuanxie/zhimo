# 资料解析与切片策略 v1.0

## 1. 目标

支持 MVP 资料导入和解析：

```text
TXT
Markdown
Word
文本型 PDF
粘贴文本
```

OCR 后置到 P1。

---

## 2. 文件支持

| 类型 | MVP | 说明 |
|---|---|---|
| TXT | 支持 | 直接读取 |
| Markdown | 支持 | 保留标题结构 |
| Word docx | 支持 | 提取正文 |
| PDF 文本型 | 支持 | 提取文本 |
| 扫描 PDF | P1 | MVP 返回 SOURCE_OCR_REQUIRED,后续迭代接入 OCR(详见技术债 TD-001) |
| HTML 网页 | P1 | 需要清洗 |

---

## 3. 解析流程

```text
创建 Source
  ↓
复制文件到项目目录
  ↓
创建 parse_source 任务
  ↓
按类型提取文本
  ↓
清洗文本
  ↓
切片
  ↓
写入 SourceChunk
  ↓
更新 Source.processing_status
```

**按类型提取文本的实现路径**:
- TXT / Markdown:前端 `SourceService.importTextFile` 直接读取文件文本
- PDF / Word:前端 `SourceService.importParsedFile` 通过 `invoke('parse_source_file')` 调用 Rust 端 `source_parser` 模块解析(详见 §12)

---

## 4. 状态流转

```text
pending
  ↓
parsing
  ↓
parsed
  ↓
ready
```

失败：

```text
parsing → failed
```

---

## 5. 文本清洗规则

需要处理：

```text
多余空行
页眉页脚
异常空格
不可见字符
连续换行
过短片段
```

不得处理：

```text
擅自改写资料原文
删除可能有意义的内容
补充不存在的内容
```

---

## 6. 切片策略

MVP 推荐：

```text
按标题优先切分
按段落补充切分
每片 600~1000 中文字符
相邻片段 overlap 100~150 字
```

SourceChunk 字段：

```text
chunk_index
content
token_count
page_number
start_offset
end_offset
```

---

## 7. Markdown 切片

优先识别：

```text
# 一级标题
## 二级标题
### 三级标题
```

标题作为 chunk metadata 保留在 content 开头。

---

## 8. Word 切片

优先提取：

```text
段落文本
标题样式，P1
列表，P1
表格，P1
```

MVP 可先按段落合并。

---

## 9. PDF 切片

文本型 PDF：

```text
按页提取文本
保留 page_number
按段落切片
```

扫描型 PDF：

```text
pdf-extract 提取文本 < 50 字符 → 判定为扫描版
返回 SOURCE_OCR_REQUIRED 错误,UI 提示"暂不支持 OCR 识别"
OCR 后置到后续迭代(详见技术债 TD-001)
```

---

## 10. 错误处理

资料解析相关的错误码统一在 `src/constants/errors.ts` 维护,UI 通过 `getErrorMessage(code)` 获取中文提示。

| 错误码 | UI 提示 | 可重试 | 触发场景 |
|---|---|---|---|
| FILE_TYPE_UNSUPPORTED | 暂不支持该文件格式 | 否 | 文件扩展名不在支持列表 |
| SOURCE_PARSE_FAILED | 资料解析失败 | 是 | Rust 端解析异常(如文件损坏、pdf-extract 报错) |
| SOURCE_OCR_REQUIRED | 检测到扫描版 PDF,暂不支持 OCR 识别 | 否 | PDF 文本 < 50 字符,判定为扫描版 |
| SOURCE_EMPTY_TEXT | 没有提取到可用文本 | 否 | 解析结果文本 trim 后为空 |
| SOURCE_OCR_MODEL_DOWNLOAD_FAILED | OCR 模型下载失败,请检查网络后重试 | 是 | 预留,当前不触发(OCR 未实现) |
| SOURCE_OCR_FAILED | OCR 识别失败 | 是 | 预留,当前不触发(OCR 未实现) |
| SOURCE_CHUNK_FAILED | 资料切片失败 | 是 | 分片入库异常 |

**预留错误码**:`SOURCE_OCR_MODEL_DOWNLOAD_FAILED` 和 `SOURCE_OCR_FAILED` 为预留错误码,当前阶段(OCR 未实现)不会触发,待后续迭代接入 OCR 后启用。

## 11. 可维护性要求

解析器按类型拆分：

```text
TextParser
MarkdownParser
DocxParser
PdfTextParser
```

统一接口：

```ts
type ParseResult = {
  rawText: string
  chunks: ParsedChunk[]
  metadata?: Record<string, unknown>
}

interface SourceParser {
  canParse(file: SourceFile): boolean
  parse(file: SourceFile): Promise<ParseResult>
}
```

> **实现说明**:MVP 阶段 PDF / Word 解析放在 Rust 端实现(见 §12),前端 `SourceService` 通过 Tauri invoke 调用。TXT / Markdown 仍走前端直接读取。上述 `SourceParser` 接口为设计参考,实际实现以 `src/services/source/SourceService.ts` 为准。

## 12. Rust 端解析模块(实现说明)

PDF 和 Word 解析因依赖原生 crate(`pdf-extract` / `docx-rs`),放在 Rust 端实现,通过 Tauri command 暴露给前端。

### 12.1 模块结构

```text
src-tauri/src/source_parser/
  ├─ mod.rs       # 入口:parse_source_file 命令,按扩展名分流
  ├─ pdf.rs       # PDF 文本提取(pdf-extract crate)
  ├─ docx.rs      # Word .docx 解析(docx-rs crate)
  └─ types.rs     # 序列化类型(ParsedSource / ParsedChunk / ParsedSourceType)
```

### 12.2 入口命令

```rust
#[tauri::command]
pub fn parse_source_file(
    app: AppHandle,
    file_path: String,
    enable_ocr: bool,
) -> Result<ParsedSource, String>
```

- 前端调用:`invoke('parse_source_file', { filePath, enableOcr: true })`
- 按 `.pdf` / `.docx` 扩展名分流到对应子模块
- 不支持的扩展名返回错误字符串

### 12.3 PDF 解析(pdf.rs)

- 使用 `pdf_extract::extract_text` 提取纯文本
- 文本 `trim().len() < 50` 判定为扫描版,返回 `PdfExtractResult::Scanned`
- 文本型 PDF 按 `\u{000C}`(form feed,PDF 分页符)切分页码,每页构造一个 `ParsedChunk`
- `page_number` 从 1 开始,`start_offset` / `end_offset` 为字符偏移
- 扫描版 PDF:当前返回 `SOURCE_OCR_REQUIRED` 错误(OCR 未实现,详见技术债 TD-001)

### 12.4 Word 解析(docx.rs)

- 使用 `docx_rs::read_docx` 读取 .docx 文件
- 遍历 `Document → Paragraph → Run → Text`,拼接段落文本
- Word 无分页概念,`page_count` 固定为 1,所有 chunk 的 `page_number = None`
- 按 `MAX_CHUNK_SIZE = 2000` 字符切分,段落间用 `\n\n` 分隔

### 12.5 类型对齐

Rust 端 `ParsedSource` 通过 `#[serde(rename_all = "camelCase")]` 序列化为 camelCase JSON,与前端 `src/types/index.ts` 的 `ParsedSource` 接口对齐:

```rust
#[derive(Serialize)]
#[serde(rename_all = "camelCase")]
pub struct ParsedSource {
    pub text: String,          // 全文文本
    pub page_count: u32,       // 总页数
    pub chunks: Vec<ParsedChunk>,
    pub source_type: ParsedSourceType,  // "pdf" | "word"
    pub ocr_used: bool,        // 是否使用 OCR(当前固定 false)
}
```

### 12.6 OCR 集成状态

OCR 集成因 `paddle-ocr-rs 0.6.1` 与 `ort 2.0.0-rc.12` 不兼容而暂停,详见技术债 TD-001。当前扫描版 PDF 返回 `SOURCE_OCR_REQUIRED` 错误,UI 提示用户"可将 PDF 转为文本型,或使用外部 OCR 工具处理后重新导入"。后续迭代恢复 OCR 的步骤见技术债文档。
