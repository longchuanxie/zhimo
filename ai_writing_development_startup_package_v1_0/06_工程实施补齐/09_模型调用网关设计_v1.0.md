# 模型调用网关设计 v1.0

## 1. 目标

统一封装 OpenAI-compatible 模型调用。

ModelGateway 负责：

```text
请求构造
API Key 解密
超时控制
取消控制
错误转换
流式输出
日志脱敏
```

---

## 2. 调用入口

业务层不得直接 fetch 模型 API。

必须调用：

```text
ModelService.callModel
  ↓
ModelGateway.callOpenAICompatible
```

---

## 3. 请求结构

```ts
type ModelCallInput = {
  provider: ModelProvider
  modelConfig: ModelConfig
  messages: ModelMessage[]
  temperature?: number
  maxOutputTokens?: number
  stream?: boolean
  signal?: AbortSignal
}
```

---

## 4. 响应结构

```ts
type ModelCallResult = {
  content: string
  raw?: unknown
  inputTokens?: number
  outputTokens?: number
  modelName: string
}
```

---

## 5. 超时策略

默认：

```text
连接超时：15 秒
普通请求超时：120 秒
长文本请求超时：300 秒
```

---

## 6. 重试策略

仅对可重试错误重试：

```text
网络抖动
429
502
503
504
```

不重试：

```text
401
403
模型不存在
参数错误
上下文过长
```

---

## 7. 取消策略

AgentRun 必须支持取消。

```ts
AbortController
```

取消后：

```text
AgentRun.status = cancelled
Task.status = cancelled
```

---

## 8. 错误映射

| 错误 | code | UI 中文 |
|---|---|---|
| API Key 错误 | MODEL_AUTH_FAILED | API 密钥无效 |
| Base URL 错误 | MODEL_ENDPOINT_FAILED | 模型服务地址不可用 |
| 模型不存在 | MODEL_NOT_FOUND | 找不到指定模型 |
| 上下文过长 | MODEL_CONTEXT_TOO_LONG | 本次参考内容过长 |
| 超时 | MODEL_TIMEOUT | 模型响应超时 |
| 速率限制 | MODEL_RATE_LIMITED | 请求过于频繁，请稍后重试 |
| 未配置模型 | MODEL_NOT_CONFIGURED | 请先配置模型服务商 |

---

## 9. 日志脱敏

禁止记录：

```text
api_key
完整 Authorization header
完整用户资料
完整上下文
```

允许记录：

```text
provider_id
model_name
status
input_tokens
output_tokens
duration_ms
error_code
```
