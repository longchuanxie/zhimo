# 错误码与中文提示表 v1.0

## 1. 目标

统一错误码、中文提示、可重试性和建议动作。

---

## 2. 通用错误

| code | 中文提示 | 可重试 |
|---|---|---|
| UNKNOWN_ERROR | 出现未知错误 | 是 |
| VALIDATION_ERROR | 输入内容不完整或格式不正确 | 否 |
| NOT_FOUND | 没有找到对应内容 | 否 |
| PERMISSION_DENIED | 当前操作不被允许 | 否 |
| OPERATION_CANCELLED | 操作已取消 | 否 |

## 4. 资料解析错误

| code | 中文提示 | 可重试 |
|---|---|---|
| SOURCE_PARSE_FAILED | 资料解析失败 | 是 |
| SOURCE_EMPTY_TEXT | 没有提取到可用文本 | 否 |
| SOURCE_OCR_REQUIRED | 检测到扫描版 PDF,暂不支持 OCR 识别 | 否 |
| SOURCE_OCR_MODEL_DOWNLOAD_FAILED | OCR 模型下载失败,请检查网络后重试 | 是 |
| SOURCE_OCR_FAILED | OCR 识别失败 | 是 |
| SOURCE_CHUNK_FAILED | 资料切片失败 | 是 |

**预留错误码说明**:`SOURCE_OCR_MODEL_DOWNLOAD_FAILED` 和 `SOURCE_OCR_FAILED` 为预留错误码,当前阶段(OCR 未实现)不会触发,待后续迭代接入 OCR 后启用。详见技术债 TD-001。

**建议动作**(部分错误码在 `src/constants/errors.ts` 中配置了 `suggestedAction` 字段,UI 可据此引导用户):
- `SOURCE_OCR_REQUIRED`:可将 PDF 转为文本型,或使用外部 OCR 工具处理后重新导入
- `SOURCE_OCR_MODEL_DOWNLOAD_FAILED`:可将模型文件手动放置到 AppData/models/ocr/ 目录

## 8. 安全要求

错误详情中不得包含：

```text
API Key
完整 Authorization Header
敏感本地路径
完整资料正文
完整 ContextPack
```
