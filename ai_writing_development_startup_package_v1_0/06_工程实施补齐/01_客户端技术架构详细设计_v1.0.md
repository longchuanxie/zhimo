# 客户端技术架构详细设计 v1.0

## 1. 架构目标

MVP 目标是构建一个本地优先的桌面客户端，保证用户在无账号、无云服务的情况下完成完整写作闭环。

核心目标：

```text
本地数据安全
开发边界清晰
AI 调用可追溯
UI 中文优先
代码可维护
```

---

## 2. 推荐技术栈

| 层级 | 技术 |
|---|---|
| 桌面壳 | Tauri 优先，Electron 备选 |
| 前端 | React + TypeScript |
| 编辑器 | TipTap / ProseMirror |
| 状态管理 | Zustand + TanStack Query |
| 数据库 | SQLite |
| 文件系统 | AppData + 项目库目录 |
| 图标 | Heroicons 24px outline |
| 模型接入 | OpenAI-compatible |
| 导出 | Markdown 原生生成，Word 使用 docx 库 |
| 任务系统 | 本地任务队列 + 事件通知 |

---

## 3. 分层架构

```text
UI Layer
  ↓
Feature Layer
  ↓
Service Layer
  ↓
Repository / Gateway Layer
  ↓
SQLite / FileSystem / Model Provider
```

---

## 4. UI Layer

职责：

- 展示中文 UI；
- 响应用户操作；
- 调用 feature hooks；
- 展示加载、空状态、错误状态；
- 不直接操作数据库；
- 不直接访问文件系统；
- 不直接调用模型接口。

禁止：

```text
组件内 SQL
组件内 fs 操作
组件内 fetch 模型 API
组件内拼接本地路径
组件内处理 API Key
```

---

## 5. Feature Layer

每个业务模块建立独立 feature：

```text
features/project
features/document
features/source
features/card
features/outline
features/knowledge
features/agent
features/context
features/model
features/export
features/task
```

职责：

- 组合 UI 组件；
- 管理页面级状态；
- 调用 Service；
- 处理业务级交互；
- 管理 Query 缓存失效。

---

## 6. Service Layer

Service 是业务边界。

```text
ProjectService
DocumentService
SourceService
CardService
OutlineService
KnowledgeService
AgentService
ContextService
ModelService
ExportService
TaskService
FileService
SecretService
```

Service 负责：

- 参数校验；
- 业务规则；
- 事务边界；
- 状态流转；
- 错误转换；
- 调用 Repository / Gateway。

---

## 7. Repository Layer

Repository 负责 SQLite 访问：

```text
ProjectRepository
DocumentRepository
SourceRepository
CardRepository
OutlineRepository
KnowledgeRepository
AgentRepository
ContextRepository
ModelRepository
ExportRepository
TaskRepository
```

Repository 不包含 UI 文案，不包含 React 状态。

---

## 8. Gateway Layer

Gateway 负责外部或系统能力：

```text
FileGateway
ModelGateway
ExportGateway
SecretGateway
LogGateway
```

## 10. 运行时主流程

### 10.1 启动流程

```text
启动客户端
  ↓
初始化 AppData 目录
  ↓
初始化 SQLite
  ↓
执行数据库迁移
  ↓
创建 default_user / default_workspace
  ↓
加载项目列表
  ↓
进入本地项目库
```

### 10.2 AI 调用流程

```text
用户触发 AI 动作
  ↓
ContextService.previewContext
  ↓
用户确认或调整本次参考内容
  ↓
ContextService.createContextPack
  ↓
AgentService.createRun
  ↓
ModelService.callModel
  ↓
AgentService.saveAssistantMessage
  ↓
用户决定采纳 / 保存为卡片 / 保存为知识
```

### 10.3 资料解析流程

```text
用户导入文件
  ↓
SourceService.createSource
  ↓
TaskService.createTask(parse_source)
  ↓
ImportService.extractText
  ↓
ChunkService.splitText
  ↓
SourceChunkRepository.bulkInsert
  ↓
Source.processing_status = ready
```

---

## 11. 可维护性要求

架构上必须保证：

```text
UI 不碰数据库
UI 不碰文件系统
UI 不碰模型 API
Service 不依赖 React
Repository 不包含业务文案
Gateway 不包含业务状态
```

任何跨层调用都必须在代码审核中指出并修正。
