# 任务系统详细设计 v1.0

## 1. 目标

统一管理本地耗时任务：

```text
资料解析
资料摘要
Agent 调用
大纲生成
导出
搜索索引构建
```

---

## 2. 任务类型

```text
parse_source
summarize_source
agent_run
generate_outline
export_document
build_search_index
```

---

## 3. 状态流转

```text
pending → running → succeeded
pending → running → failed
pending → running → cancelled
failed → pending → running
```

---

## 4. Task 数据结构

```ts
type Task = {
  id: string
  projectId?: string
  taskType: TaskType
  objectType?: string
  objectId?: string
  status: TaskStatus
  progress: number
  errorCode?: string
  errorMessage?: string
  payload?: unknown
  result?: unknown
  createdAt: string
  startedAt?: string
  completedAt?: string
}
```

---

## 5. TaskService

```ts
createTask(input): Promise<Task>
startTask(taskId): Promise<Task>
updateProgress(taskId, progress): Promise<Task>
succeedTask(taskId, result): Promise<Task>
failTask(taskId, error): Promise<Task>
cancelTask(taskId): Promise<Task>
retryTask(taskId): Promise<Task>
listTasks(projectId): Promise<Task[]>
```

---

## 6. UI 组件

```text
TaskCenter
TaskProgressItem
Toast
ErrorState
```

---

## 7. 事件通知

建议使用本地事件总线：

```text
task:created
task:progress
task:succeeded
task:failed
task:cancelled
```

前端订阅后更新任务中心和 Toast。

---

## 8. 可维护性要求

任务执行器必须按类型拆分：

```text
ParseSourceTaskRunner
AgentRunTaskRunner
ExportDocumentTaskRunner
BuildSearchIndexTaskRunner
```

禁止在 TaskService 中堆所有任务实现。
