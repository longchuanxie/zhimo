# AI 原生写作项目系统 MVP 需求文档 v0.2 定稿摘要

## 1. MVP 目标

构建一个可运行的本地写作项目客户端，支持用户完成从资料收集、资料理解、结构组织、正文写作、AI 协作到成果导出的最小闭环。

MVP 核心闭环：

```text
创建项目
  ↓
导入资料
  ↓
AI 理解资料
  ↓
生成卡片
  ↓
形成大纲
  ↓
进入编辑器写作
  ↓
智能助手基于上下文多轮协作
  ↓
AI 输出可解释建议
  ↓
用户接受 / 拒绝 / 保存为知识
  ↓
导出 Markdown / Word
```

---

## 2. 目标用户

MVP 优先支持：

```text
研究 / 论文写作者
小说 / 长文写作者
```

自由写作作为轻量模式保留。

---

## 3. 核心模块

| 模块 | MVP 要求 |
|---|---|
| 项目系统 | 创建项目、项目列表、项目首页、项目设置 |
| 文档编辑器 | 文档 CRUD、正文编辑、自动保存、字数统计、选区 AI 操作 |
| 资料库 | 本地文件导入、粘贴文本、资料解析、资料详情、AI 使用开关 |
| 卡片系统 | 手动建卡、从资料摘录成卡、卡片状态、卡片关联文档/大纲 |
| 大纲系统 | 大纲树、节点创建、节点排序、节点转文档 |
| 知识库 | 新建知识、编辑知识、从助手输出保存为知识、知识参与上下文 |
| 智能助手 | 多轮对话、绑定当前对象、快捷动作、解释建议 |
| 上下文管理 | 本次参考内容预览、ContextPack 快照、上下文排除规则 |
| 模型设置 | OpenAI-compatible 服务商配置、模型测试、任务模型配置 |
| 导出 | Markdown / Word 导出、导出任务记录、失败重试 |
| 任务中心 | 资料解析、AI 调用、导出等后台任务状态 |
| UI 规范 | 中文优先、Heroicons、空状态、错误状态 |

---

## 4. 关键约束

```text
认证功能后置
单用户本地模式
默认用户 default_user
默认工作空间 default_workspace
API Key 本地加密
AI 输出不直接覆盖正文
不展示原始思维链
禁用资料不进入上下文
已废弃知识不进入上下文
```

---

## 5. MVP 完成定义

MVP 完成时必须能跑通：

```text
打开客户端 → 创建项目 → 导入资料 → 解析资料 → 生成卡片 → 创建大纲 → 大纲节点转文档 → 写作 → 配置模型 → 调用智能助手 → 预览本次参考内容 → 保存为知识 → 导出 Word/Markdown
```
