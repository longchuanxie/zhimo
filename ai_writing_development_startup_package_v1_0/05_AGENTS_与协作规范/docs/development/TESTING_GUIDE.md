# 测试规范

## 1. 测试目标

MVP 测试必须保证：

- 客户端可启动；
- 本地数据可持久化；
- 项目闭环可跑通；
- AI 调用可追溯；
- 用户数据不丢失；
- 错误状态可恢复；
- UI 中文优先。

---

## 2. 必测路径

### 2.1 项目路径

```text
打开客户端
  ↓
创建项目
  ↓
关闭客户端
  ↓
重新打开
  ↓
项目仍存在
```

### 2.2 文档路径

```text
新建文档
  ↓
输入正文
  ↓
自动保存
  ↓
关闭并重启
  ↓
正文恢复
```

### 2.3 资料路径

```text
导入资料
  ↓
解析资料
  ↓
生成资料片段
  ↓
从片段创建卡片
```

### 2.4 Agent 路径

```text
配置模型
  ↓
打开助手面板
  ↓
预览本次参考内容
  ↓
发送消息
  ↓
生成回复
  ↓
保存为知识
```

### 2.5 导出路径

```text
选择文档
  ↓
选择格式
  ↓
创建导出任务
  ↓
生成文件
  ↓
打开文件
```

---

## 3. 错误测试

必须测试：

- 模型未配置；
- 模型连接失败；
- API Key 错误；
- 文件不存在；
- 文件无法解析；
- 数据库写入失败；
- 导出失败；
- Agent 调用超时；
- 客户端重启中断任务。

---

## 4. UI 测试

必须检查：

- 主导航为中文；
- 按钮为中文；
- 表单标签为中文；
- 状态标签为中文；
- 空状态为中文；
- 错误提示为中文；
- 图标正确显示；
- 不出现文字占位图标；
- 不向用户暴露英文对象名。

---

## 5. 数据测试

必须检查：

- created_at / updated_at 正常；
- deleted_at 软删除正常；
- project_id 关联正常；
- source_chunks 可追溯到 source；
- cards 可追溯到 source_chunk；
- knowledge 可追溯到来源；
- agent_messages 可追溯到 thread；
- agent_runs 可追溯到 model_config；
- context_packs 是快照。
