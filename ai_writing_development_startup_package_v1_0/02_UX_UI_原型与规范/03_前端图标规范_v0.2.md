# AI 原生写作项目系统前端图标规范与技术文档 v0.2

## 1. 文档信息

**文档名称：** 前端图标规范与技术实现文档  
**版本：** v0.2  
**适用范围：** 桌面客户端 MVP、空状态 UI、项目导航、功能按钮、Agent 面板、异常状态  
**产品形态：** 客户端 / Local-first / 无认证 MVP  
**默认图标库：** Heroicons  
**前端建议栈：** Tauri / Electron + React + TypeScript

---

## 2. 修订结论

前端应统一引入 Heroicons 作为默认图标库。

不再使用：

```text
文 / 资 / 卡 / 纲 / AI
```

这类文字占位图标，也不再使用临时几何块作为项目图标。

统一改为：

```text
Heroicons 24px outline icons
```

在 React 项目中统一使用：

```tsx
import {
  FolderIcon,
  DocumentTextIcon,
  ArchiveBoxIcon,
  Squares2X2Icon,
  ListBulletIcon,
  CircleStackIcon,
  ChatBubbleLeftRightIcon,
  CpuChipIcon,
  ArrowDownTrayIcon,
  MagnifyingGlassIcon,
  ExclamationTriangleIcon,
  PlusIcon,
  SparklesIcon,
  Cog6ToothIcon,
  BookOpenIcon,
} from '@heroicons/react/24/outline'
```

## 4. 图标导入规范

### 4.1 推荐导入方式

推荐：

```tsx
import { FolderIcon } from '@heroicons/react/24/outline'
```

不推荐：

```tsx
import { FolderIcon } from '@heroicons/react'
```

原因：

```text
明确使用 24px outline 版本，避免导入路径不清晰，也便于控制图标风格一致。
```

## 5. 图标尺寸规范

### 5.1 导航图标

```tsx
<FolderIcon className="h-4.5 w-4.5" />
```

建议实际 CSS：

```css
.nav-icon svg {
  width: 17px;
  height: 17px;
  stroke-width: 2;
}
```

### 5.2 按钮图标

```css
.button-icon {
  width: 17px;
  height: 17px;
  stroke-width: 2.1;
}
```

### 5.3 空状态图标

```css
.empty-icon svg {
  width: 44px;
  height: 44px;
  stroke-width: 1.5;
}
```

### 5.4 Agent 头像图标

```css
.agent-avatar svg {
  width: 22px;
  height: 22px;
  stroke-width: 2;
}
```

## 6. 图标组件封装

建议统一封装 `AppIcon`。

```tsx
import type { ComponentType, SVGProps } from 'react'

type AppIconProps = {
  icon: ComponentType<SVGProps<SVGSVGElement>>
  size?: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
  className?: string
}

const sizeClassMap = {
  xs: 'h-3.5 w-3.5',
  sm: 'h-4 w-4',
  md: 'h-5 w-5',
  lg: 'h-6 w-6',
  xl: 'h-11 w-11',
}

export function AppIcon({
  icon: Icon,
  size = 'md',
  className = '',
}: AppIconProps) {
  return <Icon className={`${sizeClassMap[size]} ${className}`} aria-hidden="true" />
}
```

### 8.2 空状态组件

```tsx
type EmptyStateProps = {
  icon: ComponentType<SVGProps<SVGSVGElement>>
  title: string
  description: string
  primaryAction?: {
    label: string
    icon?: ComponentType<SVGProps<SVGSVGElement>>
    onClick: () => void
  }
  secondaryAction?: {
    label: string
    icon?: ComponentType<SVGProps<SVGSVGElement>>
    onClick: () => void
  }
  hint?: string
}
```

示例：

```tsx
<EmptyState
  icon={ArchiveBoxIcon}
  title="还没有导入资料"
  description="你可以导入 PDF、Word、Markdown、TXT，或直接粘贴文本。资料默认保存在本机。"
  primaryAction={{
    label: '导入文件',
    icon: ArchiveBoxIcon,
    onClick: openFilePicker,
  }}
  secondaryAction={{
    label: '粘贴文本',
    icon: DocumentTextIcon,
    onClick: openPasteTextModal,
  }}
  hint="可将文件拖拽到此区域。扫描件 OCR 后置到 P1。"
/>
```

## 10. 颜色与风格约束

### 10.1 图标颜色

| 场景 | 颜色 |
|---|---|
| 默认导航 | `--muted` |
| 当前导航 | `--brand` |
| 空状态主图标 | `--brand` |
| AI 快捷动作 | `--brand` |
| 警告 | `--warning` |
| 错误 | `--danger` |
| 次级说明 | `--subtle` |

### 10.2 禁止事项

不要使用：

```text
emoji 作为正式图标
文字缩写作为图标
无语义几何块作为图标
多个图标库混用导致风格不一致
彩色 3D 图标
强 AI 感发光图标
```

## 12. 原型到前端实现说明

本次 HTML 原型使用内联 SVG symbol 模拟 Heroicons 24/outline 的视觉规范，以保证文件可离线打开。

前端真实实现时应替换为：

```tsx
@heroicons/react/24/outline
```

不要直接复制原型中的 SVG symbol 作为生产代码。

## 13. 验收标准

### 13.1 UI 验收

- 所有导航项必须使用 SVG 图标；
- 所有空状态必须有主图标；
- 所有主 CTA 必须有动作图标；
- Agent 快捷动作必须使用 `SparklesIcon` 或语义图标；
- 错误状态必须使用 `ExclamationTriangleIcon`；
- 不允许文字占位图标。

### 13.2 技术验收

- 项目已安装 `@heroicons/react`；
- 图标统一从 `/24/outline` 导入；
- 图标通过 `AppIcon` 或统一组件封装；
- 业务对象与图标映射集中维护；
- 不允许在业务页面中散落硬编码 SVG；
- 图标颜色跟随设计 token，不单独写死。

---

## 14. 版本结论

v0.2 起，AI 原生写作项目系统的客户端 UI 不再使用临时图标方案。

统一采用：

```text
Heroicons 24px outline
```

并通过：

```text
业务对象 → 图标映射 → AppIcon 组件 → 页面使用
```

形成稳定的图标规范。

这可以让项目在 MVP 阶段就具备统一、克制、专业的桌面客户端视觉气质。
