# 空状态设计规范 v0.1

## 1. 空状态目标

空状态不是“暂无数据”，而是引导用户进入下一步动作。

核心原则：

```text
解释当前页面价值
降低第一次操作成本
引导用户进入核心闭环
```

---

## 2. 标准结构

```text
图标
标题
说明文字
主行动按钮
次行动按钮
辅助提示
研发映射
```

---

## 3. 空状态总表

| 页面 | 标题 | 主按钮 |
|---|---|---|
| 项目列表 | 还没有写作项目 | 创建项目 |
| 项目首页 | 这个项目还很空 | 导入资料 |
| 文档编辑 | 从这里开始写作 | 起草开头 |
| 资料库 | 还没有导入资料 | 导入文件 |
| 卡片库 | 还没有卡片 | 新建卡片 |
| 大纲页 | 还没有大纲 | AI 生成大纲 |
| 知识库 | 还没有项目知识 | 新建知识 |
| 助手对话 | 还没有助手对话 | 去文档中开始 |
| 模型设置 | 还没有配置模型 | 添加模型服务商 |
| 导出页 | 还没有可导出的内容 | 新建文档 |
| 搜索无结果 | 没有找到匹配结果 | 清除搜索 |
| 筛选无结果 | 当前筛选下没有内容 | 清除筛选 |

---

## 4. 客户端提示

空状态中可提示：

```text
数据优先保存在本机
无需登录即可开始
API Key 加密保存在本机
资料可控制是否允许 AI 使用
```
