# AI 原生写作项目系统 UI 缺失组件细化清单 v0.1

## 1. 文档目的

当前 UI 原型已经覆盖：

- 空状态
- 项目入口
- 文档编辑页
- 资料库
- 卡片库
- 大纲
- 知识库
- Agent 面板
- 模型设置
- 导出

但仍缺少大量前端实际开发所需的可复用组件。

本文件用于补充这些组件，形成前端组件拆分依据。

---

# 2. 组件分层

建议按以下层级拆分：

```text
基础组件
  ↓
布局组件
  ↓
业务对象组件
  ↓
Agent / Context 组件
  ↓
任务与反馈组件
  ↓
页面组合组件
```

# 16. 结论

当前 UI 原型已经完成页面方向和空状态方向，但前端真正落地还缺少组件层设计。

下一步前端应优先实现：

```text
基础组件 → 布局组件 → 业务对象列表组件 → Agent / Context 组件 → 任务反馈组件
```

其中最关键的是：

```text
ContextPackPreview
AgentOutputActions
AIDiffPreview
SelectionFloatingMenu
SourceParseProgress
TaskCenter
```

这些决定产品是否只是“好看的页面”，还是一个真正可用的 AI 写作客户端。
