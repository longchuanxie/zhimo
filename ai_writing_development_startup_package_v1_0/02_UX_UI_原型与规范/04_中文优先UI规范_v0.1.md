# AI 原生写作项目系统中文优先 UI 规范 v0.1

## 1. 修订结论

UI 原型和前端界面应以中文为第一显示语言。

用户可见内容包括：

- 页面标题
- 导航名称
- 按钮文案
- 表单标签
- 空状态文案
- Agent 面板文案
- 状态标签
- Toast 通知
- 错误提示
- 任务名称
- 导出向导
- 模型设置页

均应优先使用中文。

英文仅保留在：

- 数据库字段
- TypeScript 类型
- Service / API 名称
- 调试信息
- 研发映射说明
- 第三方协议或模型名称

# 3. 核心对象中文命名表

| 研发对象 | UI 中文名称 | 备注 |
|---|---|---|
| Project | 项目 | 不显示 Project |

| Source | 资料 | 不建议叫“来源” |

| Knowledge | 知识 | 页面名用“知识库” |
| AgentThread | 助手对话 | 不直接显示 AgentThread |
| AgentMessage | 助手消息 | 一般无需显示 |
| AgentRun | 助手任务 | 或“生成任务” |
| ContextPack | 上下文包 / 本次参考内容 | 用户侧优先用“本次参考内容” |
| ModelProvider | 模型服务商 | 设置页使用 |

```text

```

# 13. 验收标准

## 13.1 UI 文案验收

- 主导航全部中文；
- 页面标题全部中文；
- 按钮全部中文；
- 状态标签全部中文；
- 空状态全部中文；
- 表单 label 全部中文；
- Agent 面板用户可见文案全部中文；
- 错误提示全部中文；
- Toast 全部中文；
- 任务中心全部中文。

## 13.2 允许英文的情况

- API 名称；
- Service 名称；
- 数据库字段；
- 枚举值；
- 模型名称；
- 第三方服务名称；
- 研发映射说明；
- 调试模式。

---

# 14. 版本结论

从 v0.2 起，AI 原生写作项目系统的 UI 原型必须采用：

```text
中文优先，英文辅助
```

即：

```text
用户看到中文
研发看到映射
数据仍保持英文结构
```

这可以保证产品首先适配中文写作者，同时不影响前后端、数据库和本地服务层的工程实现。
