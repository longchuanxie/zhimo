## 二、设计原则与关键决策

### 2.1 论文写作四项严格要求的落地策略

**① 内容格式**
- 章节编号：导出阶段统一通过 `injectSectionNumbers` 共用工具，向 TipTap JSON 的 heading 节点注入 `attrs.numberPrefix`（如 "1.1"），LaTeX/DOCX/PlainText 序列化器统一消费，避免编号逻辑分散。
- 引用格式：MVP 锁定 GB/T 7714-2015（已有 `Gbt7714Formatter`），导出向导可选 `citationStyle`。
- 文档结构：LaTeX 导出含完整 preamble（`\documentclass{article}` + 中文支持 `\usepackage{ctex}` + 图片/公式宏包 + `\bibliography`）；DOCX 导出含标题层级 + 参考文献节。

**② 公式**
- 块级公式：编辑器内 KaTeX 实时预览（`EquationBlockView` 已实现）；入库时 `validateLatex` 强制校验语法；导出时 LaTeX 原样输出 `\begin{equation}`，DOCX 降级为 Cambria Math 字体文本（技术债 TD-PAPER-002，不实现 MathML→PNG）。
- 行内公式：TipTap `Mathematics` 扩展已在编辑器加载，LaTeX 序列化为 `$...$`，DOCX 序列化为 `$...$` 文本。
- label 唯一性：`EquationService.createEquation` 已校验文档内唯一；`PaperService.checkIntegrity` 兜底检测重复。

**③ 图表插入**
- figure/table 独立编号序列（`FigureService` 已实现）。
- caption 必填（`FigureService.createFigure` 已校验）；`checkIntegrity` 兜底检测缺失。
- 材料真实性：figure 可关联 `sourceId`，`checkIntegrity` 对缺失 source 发出 `missing_source` 警告（不阻断，仅提示）。
- 导出：LaTeX 输出 `\begin{figure}\includegraphics\caption`；DOCX 输出 `ImageRun`（figure）或 `Table`（table）+ 题注段落。
- 交叉引用：`CrossReference` mark 通过 label 关联，导出时 LaTeX 输出 `\ref{}`，DOCX 输出编号文本。

**④ 材料真实性（最严格）**
- 保障链：`Source`（本地导入资料）→ `BibliographicMetadata`（书目元数据）→ `Reference`（参考文献库，`sourceId` 可溯源）→ `Citation`（文档内引文实例，`referenceId` 强关联）→ `PaperService.checkIntegrity` 检测 `orphan_citation`。
- **导出前阻断**：仅 `orphan_citation`（引文关联的参考文献已删除）阻断导出并返回 `CITATION_ORPHAN` 错误；其余完整性 issue（`missing_caption`/`missing_source`/`label_duplicate`/`invalid_latex`）为警告，不阻断但通过 `PaperIntegrityPanel` 提示用户。
- Agent 校对：新增论文校对 prompt，Agent 可基于 ContextPack 中的参考文献 + 图表 + 公式清单，检查引文规范性和材料可溯源性。
- `SourcePickerModal`：创建 Reference / Figure 时，强制引导用户从已导入资料中选择来源（可跳过，但跳过时记录技术债）。

### 2.2 技术债记录（MVP 取舍）

| 编号 | 内容 | 原因 | 后续修复 |
|---|---|---|---|
| TD-PAPER-002 | DOCX 公式以 LaTeX 源码文本输出 | MathML→PNG 转换复杂，MVP 不实现 | 后续接入 `mathjax-node` 生成 PNG |
| TD-PAPER-006 | DOCX 脚注以 `[脚注: content]` 文本输出 | `docx` 库 Footnote API 较复杂 | 后续使用 `docx.Footnotes` |
| TD-PAPER-007 | DOCX 交叉引用以编号文本输出 | 无动态字段支持 | 后续使用 Word 域代码 |

## 三、实施步骤（按依赖顺序）

### 阶段 A：序列化器收尾

#### A.1 新建 `src/components/editor/editorToDocx.ts`
**职责**：TipTap JSON → `docx.Document` 对象，供 `DocxExporter` 生成真实 `.docx` 二进制。
**依赖**：`docx@^9.7.1`（已安装）。
**签名**：
```ts
import type { JSONContent } from '@tiptap/react'
import { Document, Paragraph, TextRun, HeadingLevel, AlignmentType } from 'docx'
import type { Reference, Figure, Equation, Citation } from '@/types'

export type EditorToDocxOptions = {
  references: Reference[]
  figures: Figure[]
  equations: Equation[]
  citations: Citation[]
  figureImageBuffers?: Map<string, Uint8Array> // figureId → 图片二进制
}

export function editorToDocx(
  content: JSONContent | null,
  options: EditorToDocxOptions,
): Document
```
**转换规则**：
- `doc` → `Document({ sections: [{ children: Paragraph[] }] })`
- `heading` level 1/2/3 → `Paragraph({ heading: HeadingLevel.HEADING_1/2/3, children: [TextRun({ text, bold: true })] })`，前缀 `attrs.numberPrefix`
- `paragraph` → `Paragraph({ children: TextRun[] })`（行内 marks 合并到 TextRun 的 `bold`/`italic`/`font` 属性）
- `blockquote` → `Paragraph({ indent: { left: 720 }, children: [...] })`
- `bulletList`/`orderedList` → 每项 `Paragraph({ bullet: { level: 0 } })` 或 ` numbering: { reference: 'ordered', level: 0 } }`
- `codeBlock` → `Paragraph({ children: [TextRun({ text, font: 'Consolas' })] })`
- `horizontalRule` → `Paragraph({ border: { bottom: { style: BorderStyle.SINGLE } } })`
- `figureBlock`(kind=figure) → 若有 `figureImageBuffers` 则 `Paragraph({ children: [ImageRun({ data, transformation: { width, height } })] })`，否则文本占位 `[图片: caption]`；后接 `Paragraph({ alignment: CENTER, children: [TextRun({ text: '图 N caption', italics: true })] })`
- `figureBlock`(kind=table) → `Table({ rows: [...] })` + 题注段落
- `equationBlock` → `Paragraph({ alignment: CENTER, children: [TextRun({ text: latex, font: 'Cambria Math' })] })`（TD-PAPER-002）；后接编号 `(N)`
- `table`(TipTap) → `docx.Table`
- `image` → `ImageRun`
- 行内 marks：
  - `bold`/`italic`/`code` → TextRun 属性
  - `citation` → `TextRun({ text: citation.inlineText || '[?]' })`
  - `crossReference` → `TextRun({ text: '见图 N' })`（TD-PAPER-007）
  - `footnote` → `TextRun({ text: `[脚注: ${content}]` })`（TD-PAPER-006）
  - `mathematics` → `TextRun({ text: \`$${text}$\` })`
- 文档末尾追加"参考文献"节：每个 Reference 一行 `Paragraph({ children: [TextRun({ text: gbt7714Formatted })] })`

#### A.2 改造 `src/components/editor/editorToPlainText.ts`
**现状**：`extractText` 对未知节点返回 `childText`，但 `figureBlock`/`equationBlock` 无文本子节点，返回空。
**改造**：在 `extractText` 中新增分支：
- `figureBlock` → `[图N: caption]` 或 `[表N: caption]`（从 attrs 读取 figureId，但纯文本函数无 options，简化为从 attrs.caption 或 attrs.label 输出占位）。实际方案：保留纯函数签名，对 `figureBlock` 输出 `\n[图表]\n`，对 `equationBlock` 输出 `\n[公式]\n`，对 `citation` mark 输出其 `inlineText`，对 `crossReference` mark 输出 `[引用]`，对 `footnote` mark 输出 `[脚注]`。

### 阶段 B：导出增强

#### B.1 新建 `src/services/export/injectSectionNumbers.ts`
**职责**：遍历 TipTap JSON，为 heading 节点注入 `attrs.numberPrefix`（如 "1"、"1.1"、"1.1.1"），供所有序列化器统一消费。
**签名**：
```ts
import type { JSONContent } from '@tiptap/react'

export function injectSectionNumbers(content: JSONContent): JSONContent
```
**规则**：维护 `counters: [number, number, number]`，遇 level 1 重置 counters[1]/[2] 并递增 counters[0]，输出 `${counters[0]}`；遇 level 2 递增 counters[1] 并重置 counters[2]，输出 `${counters[0]}.${counters[1]}`；遇 level 3 递增 counters[2]，输出 `${counters[0]}.${counters[1]}.${counters[2]}`。深拷贝后写入 `node.attrs.numberPrefix`。

#### B.2 新建 `src/services/export/LatexExporter.ts`
**职责**：生成完整 `.tex` 文件源码（preamble + body + bibliography）。
**签名**：
```ts
import type { JSONContent } from '@tiptap/react'
import type { Reference, Figure, Equation, Citation, ExportOptions } from '@/types'

export type LatexExportInput = {
  title: string
  content: JSONContent
  references: Reference[]
  figures: Figure[]
  equations: Equation[]
  citations: Citation[]
  options: ExportOptions
}

export function exportToLatex(input: LatexExportInput): string
```
**输出结构**：
```latex
\documentclass{article}
\usepackage{ctex}
\usepackage{graphicx}
\usepackage{amsmath}
\usepackage{hyperref}
\title{...}
\begin{document}
\maketitle
\tableofcontents  % 若 options.includeTOC
% body（经 injectSectionNumbers + editorToLatex）
\section{...}
...
% 参考文献
\begin{thebibliography}{99}
\bibitem{key1} GB/T 7714 格式条目
\end{thebibliography}
\end{document}
```
**实现**：调用 `injectSectionNumbers(content)` → `editorToLatex(numbered, {references,figures,equations,citations})` → 拼接 preamble + bibliography（复用 `Gbt7714Formatter`）。

#### B.3 新建 `src/services/export/DocxExporter.ts`

## 五、假设与决策

1. **MVP 引用格式**：仅实现 GB/T 7714-2015，UI 预留其他选项但禁用。
2. **图片存储**：MVP 使用 base64 内联（`figure.imageData`），不实现本地文件复制；导出 DOCX 时从 base64 解码为 Uint8Array。
3. **LaTeX 导出图片**：`figure.imagePath` 为空时输出占位 `image-placeholder`，不实际复制图片文件（技术债）。
4. **多文档导出**：按文档顺序拼接，LaTeX 用 `\section{文档标题}` 分隔，DOCX 用分页符。
5. **Agent 论文校对**：复用现有 `format_text` taskType，通过 boundObject=document 判断是否使用论文 prompt，不新增 taskType。
6. **EditorModals 位置**：挂载在 Editor 组件内（非全局），因为所有插入操作需要 editor 实例。
7. **完整性面板刷新**：debounce 2 秒，避免编辑时频繁查询数据库。
