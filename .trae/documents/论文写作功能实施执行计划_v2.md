### 3.1 新增迁移 `006_relax_export_format_check.sql`

**目标**:通过 SQLite 标准重建表流程,放开 `export_format` CHECK 约束为 `('markdown','word','latex','docx')`,数据零丢失。

**文件**:`src-tauri/migrations/006_relax_export_format_check.sql`

**SQL 逻辑**(遵循 SQLite 重建表 12 步法):
```sql
-- 006: 放开 export_tasks.export_format CHECK 约束,支持 latex/docx
-- SQLite 不支持 ALTER CHECK,通过重建表实现

BEGIN TRANSACTION;

-- 1. 创建新表(放宽 CHECK)
CREATE TABLE export_tasks_new (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  export_scope TEXT NOT NULL,
  export_format TEXT NOT NULL CHECK (export_format IN ('markdown','word','latex','docx')),
  document_ids TEXT,
  outline_node_ids TEXT,
  file_path TEXT,
  status TEXT NOT NULL DEFAULT 'pending',
  error_code TEXT,
  error_message TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  completed_at TEXT,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 2. 复制数据(含 005 新增的 export_options 列)
INSERT INTO export_tasks_new (id, project_id, export_scope, export_format, document_ids, outline_node_ids, file_path, status, error_code, error_message, created_at, completed_at)
SELECT id, project_id, export_scope, export_format, document_ids, outline_node_ids, file_path, status, error_code, error_message, created_at, completed_at
FROM export_tasks;

-- 3. 删除旧表
DROP TABLE export_tasks;

-- 4. 重命名新表
ALTER TABLE export_tasks_new RENAME TO export_tasks;

-- 5. 重建索引
CREATE INDEX IF NOT EXISTS idx_export_tasks_project ON export_tasks(project_id, created_at);

COMMIT;
```

### 4.2 新增文件清单

#### 4.2.1 `src/services/citation/Gbt7714Formatter.ts` — GB/T 7714-2015 格式化器
- **类型**:纯函数模块,无副作用,无 DB 访问
- **输入**:`Reference` + 可选 `CitationStyle`(MVP 仅实现 gbt7714_2015)
- **输出**:格式化字符串
- **核心函数**:
  - `formatReference(reference: Reference): string` — 按 entryType 分派
  - `formatAuthors(authors, isChinese): string` — 超过 3 人显示前 3 + ", 等"/", et al."
  - `formatInlineText(citation, reference, style): string` — 生成 '[1]' / '(Smith, 2020)'
- **格式规范**(严格遵循国标):
  - 期刊 [J]:`作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.`
  - 专著 [M]:`作者. 书名[M]. 版次. 出版地: 出版者, 年: 起止页码.`
  - 会议 [C]:`作者. 题名[C]//会议名. 出版地: 出版者, 年: 起止页码.`
  - 学位论文 [D]:`作者. 题名[D]. 保存地: 保存单位, 年.`
  - 电子文献 [EB/OL]:`作者. 题名[EB/OL]. (发布日期)[引用日期]. URL.`
  - 其他 [Z]:`作者. 题名[Z]. 出版地: 出版者, 年.`
- **作者检测中英文**:通过作者名是否含 CJK 字符判断

#### 4.2.2 `src/services/citation/ReferenceService.ts` — 参考文献库管理
- **遵循**:[CardService.ts](file:///d:/workplace/idea/zhimo/src/services/card/CardService.ts) 模式,返回 `ServiceResult<T>`,使用 `ok/err/fromUnknown`
- **方法**:
  - `listReferences(projectId): ServiceResult<Reference[]>`
  - `getReference(id): ServiceResult<Reference>` — 未找到返回 `REFERENCE_NOT_FOUND`
  - `createReference(input): ServiceResult<Reference>` — 校验 citationKey 项目内唯一,重复返回 `CITATION_KEY_DUPLICATE`;校验必填字段(title/authors/entryType)
  - `updateReference(id, patch): ServiceResult<Reference>`
  - `deleteReference(id): ServiceResult<void>` — 软删除前检查是否有 citation 引用,有则返回警告信息(不阻断,由 UI 决定)
  - `importFromSource(sourceId): ServiceResult<Reference>` — 读取 source.bibliographicMetadata,生成 reference,sourceId 自动关联(材料真实性核心);元数据无效返回 `BIBLIOGRAPHIC_METADATA_INVALID`

#### 4.2.3 `src/services/citation/CitationService.ts` — 引文实例管理
- **方法**:

  - `renumberCitations(documentId): ServiceResult<void>` — 按 prosemirrorPos 升序重新分配 numeric 编号(1,2,3...)

- **依赖**:citationRepository、referenceRepository、Gbt7714Formatter

#### 4.2.4 `src/services/figure/FigureService.ts` — 图表管理
- **方法**:
  - `listFiguresByDocument(documentId, kind?): ServiceResult<Figure[]>`
  - `listFiguresByProject(projectId, kind?): ServiceResult<Figure[]>`
  - `createFigure(input): ServiceResult<Figure>` — 自动分配 number(调用 figureRepository.getMaxFigureNumber + 1,按 kind 独立序列);校验 caption 非空;校验 label 项目内不冲突

#### 4.2.5 `src/services/equation/EquationService.ts` — 公式管理
- **方法**:
  - `listEquationsByDocument(documentId): ServiceResult<Equation[]>`

#### 4.2.6 `src/services/paper/PaperService.ts` — 论文编排协调层
- **职责**:跨 Service 协调,扫描 TipTap JSON 重建索引,完整性检查
- **方法**:
  - `rebuildDocumentIndex(documentId): ServiceResult<void>` — 读取 document.contentJson,递归扫描 CitationMark/FigureBlock/EquationBlock 节点,同步 citations/figures/equations 表(新增/更新位置/删除已移除的)
  - `resolveAllCrossReferences(documentId): ServiceResult<void>` — 扫描 CrossReference mark,根据 targetType + targetId 解析真实编号,更新 displayText;目标不存在标记为悬空
  - `checkDocumentIntegrity(documentId): ServiceResult<IntegrityIssue[]>` — 聚合检查:① 悬空引用(CitationService.checkOrphanCitations)② 悬空交叉引用(FigureService.checkOrphanCrossRefs)③ 缺失 caption(figure.caption 为空)④ 缺失 sourceId(figure 无来源,警告级)⑤ 编号冲突 ⑥ label 重复 ⑦ 无效 LaTeX(EquationService.validateLatex)
  - `getDocumentPaperMeta(documentId): ServiceResult<DocumentPaperMeta>` — 聚合返回 {citationCount, figureCount, tableCount, equationCount, wordCount, integrityIssueCount}
- **依赖**:citationService、figureService、equationService、referenceService、documentRepository

### 5.5 序列化器(新建 `src/components/editor/serializers/`)

#### 5.5.1 `editorToLatex.ts`
- **输入**:TipTap JSON + 文档元数据
- **输出**:LaTeX 源码字符串
- **递归转换规则**:
  - doc → 递归子节点
  - heading(level 1-6) → `\section{}` / `\subsection{}` ... + `\label{sec:xxx}`
  - paragraph → 段落 + 空行分隔
  - blockquote → `\begin{quote}...\end{quote}`
  - bold/italic → `\textbf{}`/`\textit{}`
  - code(inline) → `\texttt{}`
  - codeBlock → `\begin{verbatim}...\end{verbatim}`
  - bulletList/orderedList → `\begin{itemize}`/`\begin{enumerate}`
  - Mathematics inline → 透传 `$...$`
  - EquationBlock → `\begin{equation}\label{eq:xxx} latex \end{equation}`
  - FigureBlock(image) → `\begin{figure}[htbp]\centering\includegraphics[width=0.8\textwidth]{path}\caption{caption}\label{fig:xxx}\end{figure}`
  - FigureBlock(table) → `\begin{table}[htbp]\centering\caption{caption}\label{tbl:xxx}\begin{tabular}{...}...\end{tabular}\end{table}`
  - CitationMark → `\cite{citationKey}`
  - CrossReference → `\ref{label}`
  - Footnote → `\footnote{text}`
  - horizontalRule → `\hrulefill`
- **特殊字符转义**:`& % $ # _ { } ~ ^ \` → `\&` `\%` 等(公式节点内不转义)
- **文档骨架**:
  ```latex
  \documentclass[12pt,a4paper]{article}
  \usepackage[UTF8]{ctex}
  \usepackage{amsmath,amssymb}
  \usepackage{graphicx}
  \usepackage{booktabs}
  \usepackage{hyperref}
  \usepackage{url}
  \title{...}\author{...}\date{...}
  \begin{document}\maketitle
  ... 正文 ...
  \begin{thebibliography}{99}
  \bibitem{key1} 格式化字符串1
  \bibitem{key2} 格式化字符串2
  \end{thebibliography}
  \end{document}
  ```
- **参考文献**:调用 CitationService.generateBibliography 输出

#### 5.5.2 `editorToDocx.ts`
- **输入**:TipTap JSON + ExportOptions
- **输出**:`Uint8Array`(docx 二进制,通过 `docx` npm 包 `Packer.toBuffer`)
- **转换规则**:使用 `docx` 包的 `Document`/`Paragraph`/`TextRun`/`Table`/`ImageRun`/`Header`/`Footer` API
- **公式处理**:MVP 方案——LaTeX → KaTeX 渲染 MathML → 转 OMML;复杂公式降级为 KaTeX 渲染 SVG 插入 ImageRun
- **样式应用**:ExportOptions.fontFamily → `RunFonts`,fontSize → `size`(half-points),lineHeight → `Paragraph.spacing.line`,margin → `PageMargin`
- **页眉页脚**:页眉显示文档标题,页脚显示页码
- **参考文献**:末尾 `Table` 或 `Paragraph[]`,调用 CitationService.generateBibliography

## 六、阶段 4:导出增强

### 6.1 新增 `src/services/export/LatexExporter.ts`
- **导出函数**:`generateLatex(documents: Document[], options: ExportOptions): Promise<string>`
- **依赖**:editorToLatex(序列化)、CitationService.generateBibliography(参考文献)
- **流程**:
  1. 遍历 documents,对每个 doc.contentJson 调用 editorToLatex 生成正文片段
  2. 拼接文档头(\documentclass + 包引入)
  3. 拼接标题/作者/日期
  4. 拼接正文
  5. 调用 CitationService.generateBibliography 生成 \begin{thebibliography}
  6. 拼接 \end{document}
- **错误处理**:LaTeX 语法错误返回 `LATEX_SYNTAX_INVALID`,序列化失败返回 `EXPORT_LATEX_FAILED`

### 6.2 新增 `src/services/export/DocxExporter.ts`
- **导出函数**:`generateDocx(documents: Document[], options: ExportOptions): Promise<Uint8Array>`
- **依赖**:editorToDocx(序列化)、docx npm 包、CitationService.generateBibliography
- **流程**:
  1. 遍历 documents,对每个 doc.contentJson 调用 editorToDocx 生成 Paragraph[]/Table[]
  2. 应用 ExportOptions 样式(fontFamily/fontSize/lineHeight/margin)
  3. 添加页眉(文档标题)/页脚(页码)
  4. 末尾添加参考文献 Paragraph[]
  5. 调用 `new Document({sections})` + `Packer.toBuffer(doc)` 返回 Uint8Array
- **错误处理**:生成失败返回 `EXPORT_DOCX_FAILED`
