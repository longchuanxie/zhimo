## 3. 提议变更

### 3.1 抽取 `FormField` / `TypeOption` 为共享组件

**Why**：AGENTS.md §3.1 要求"低重复"。新页面与 `CreateProjectPage` 都需要这两个组件，复制会违反红线。

**What**：
- 新建 `src/features/project/components/FormField.tsx`：导出 `FormField` 组件（含 `label`/`required`/`hint`/`children` props）
- 新建 `src/features/project/components/TypeOption.tsx`：导出 `TypeOption` 组件（含 `type`/`selected`/`onClick` props）
- 修改 `src/features/project/CreateProjectPage.tsx`：删除内部 `FormField`/`TypeOption` 定义，改为 `import { FormField } from './components/FormField'` 与 `import { TypeOption } from './components/TypeOption'`

**How**：
- 组件代码原样搬移，不修改逻辑
- props 类型定义同步迁移到组件文件
- `CreateProjectPage.tsx` 的 import 路径用相对路径 `./components/FormField`
- `PROJECT_TYPE_LABEL` 在 `TypeOption.tsx` 内部 import（原 `CreateProjectPage.tsx` 也是从 `@/constants/status` 引入）

### 3.2 新建 `CreateProjectFromDocumentPage.tsx`

**路径**：`src/features/project/CreateProjectFromDocumentPage.tsx`

**Why**：UI 入口，承接 `IMPORT_STEPS` 定义的三步流程，调用已就绪的三个 Service 函数。

**What**：单页面三步流程组件，步骤间用状态机切换，不拆多路由。

**组件结构**（控制单组件 ≤250 行，超阈值则拆子组件）：

```
CreateProjectFromDocumentPage (主组件, 状态机 + 顶部导航)
├── DocumentImportStepSelect   (Step 1：选择文档)
├── DocumentImportStepInfer    (Step 2：AI 推断元数据)
└── DocumentImportStepConfirm  (Step 3：确认创建表单)
```

**状态管理**（主组件 useState）：
```ts
type Step = 'select' | 'infer' | 'confirm'
type PageState = {
  step: Step
  documentPath: string
  structuredDoc: StructuredDoc | null
  inferredMeta: InferredProjectMeta | null
  documentTitle: string
  errorMsg: string | null
  loading: boolean
  submitting: boolean
}
```

**Step 1 选择文档**（`DocumentImportStepSelect`）：
- 一个「选择文档」按钮 → 调 `pickAndParseDocument()`
- 成功后显示文档信息卡：格式标签（`IMPORT_FORMAT_LABEL[doc.format]`）、字数、节点数
- 「下一步」按钮进入 Step 2
- 用户取消选择（`OPERATION_CANCELLED`）静默忽略，不报错
- 其他错误显示在卡片下方

**Step 2 推断元数据**（`DocumentImportStepInfer`）：
- 进入时自动调 `inferProjectFromDocument(doc.plainText)`
- 加载态：spinner + "AI 正在分析文档..."
- 成功：显示推断结果摘要（名称、类型、描述、目标字数），「下一步」进入 Step 3
- 失败（`PROJECT_INFERENCE_FAILED`）：显示错误 + 「手动填写」按钮，点击后用空 meta 进入 Step 3
- 失败（`MODEL_NOT_CONFIGURED`）：显示错误 + 「去配置模型」按钮（导航到 `/settings/models`）+ 「手动填写」按钮

**Step 3 确认创建**（`DocumentImportStepConfirm`）：
- 表单复用 `FormField`/`TypeOption`，字段：name（required）、type、description、writingGoal、targetReader、targetWordCount、documentTitle（默认 `IMPORT_DEFAULT_DOCUMENT_TITLE`）
- 表单初始值来自 `inferredMeta`（若为 null 则空表单）
- 「创建项目」按钮 → 调 `createProjectFromDocument({ documentPath, structuredDoc, meta, documentTitle })`
- 成功：`navigate(/projects/${projectId}, { replace: true })`
- 失败：显示错误（`DOCUMENT_IMPORT_FAILED` 等可重试）

**顶部导航**：返回按钮（select 步返回 `/projects`，其他步返回上一步）+ 标题「从文档创建项目」+ 步骤指示器（1/2/3）

**错误处理**：
- 错误码统一用 `result.error.message`（已在 `errors.ts` 定义中文提示）
- 不在组件内硬编码错误文案
- 加载/错误/空态全覆盖（AGENTS.md §4.6）

**图标**（AGENTS.md §8 强制 Heroicons outline）：
- 选择文档：`ArrowUpTrayIcon`
- 下一步：`ArrowRightIcon`
- 创建：`CheckIcon`
- 返回：`ArrowLeftIcon`
- 文档信息：`DocumentTextIcon`

## 4. 假设与决策

### 4.1 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| `FormField`/`TypeOption` 复用方式 | 抽到 `src/features/project/components/` 共享 | AGENTS.md §3.1 低重复原则；两个组件都很简单，抽出后 `CreateProjectPage` 改动最小（仅 import 路径） |
| 页面流程模式 | 单页三步状态机 | 对齐 `IMPORT_STEPS` 设计；参考 `CreateProjectPage` 也是单页表单；拆多路由会增加路由复杂度 |
| AI 推断失败兜底 | 允许手动填写空表单进入 Step 3 | `PROJECT_INFERENCE_FAILED.suggestedAction` 已暗示"可手动填写项目信息后继续" |
| 路由路径 | `projects/new-from-document` | 与 `projects/new-guided` 命名一致 |
| 入口图标 | `ArrowUpTrayIcon` | Heroicons outline 上传图标，语义贴合"导入文档" |
| 空状态入口 | 暂不调整 | 现有 `EmptyState` 已有 primary/secondary，新增 tertiary 会破坏布局；顶部操作栏入口已足够 |
| Step 2 自动触发推断 | 进入步骤即调 `inferProjectFromDocument` | 减少用户操作；失败有兜底 |

### 4.2 假设

- `pickAndParseDocument` 内部 `open()` 在用户取消时返回 `null`，已处理为 `OPERATION_CANCELLED`（无需组件层特殊处理）
- `createProjectFromDocument` 失败不回滚是已知技术债（TD-IMPORT-03），本计划不修复
- `InferredProjectMeta.targetWordCount` 是 number 类型，表单中需转为 string 显示，提交时转回 number
- TipTap 转换器已测试通过，UI 层无需重复测试转换逻辑
- `DocumentService.createDocument` 签名与 `CreateProjectPage` 调用方式一致（`{ projectId, title }`）

### 4.3 不做的事（避免过度工程）

- 不抽 `callInferenceModel`/`safeParseJson`/`normalizeProjectType` 公共 util（TD-IMPORT-05 留待后续）
- 不实现 `createProjectFromDocument` 失败回滚（TD-IMPORT-03 留待后续）
- 不实现 docx 图片占位节点处理（TD-IMPORT-02 留待后续）
- 不为 `CreateProjectFromDocumentPage` 写组件级单测（Service 层已有覆盖，UI 测试成本高收益低，符合 MVP 取舍）
- 不调整 `ProjectListPage` 空状态布局
