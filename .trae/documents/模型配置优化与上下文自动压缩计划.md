# 模型配置优化与上下文自动压缩 优化计划

## 概述

本计划针对两个问题进行优化：
1. **模型配置页面**：maxOutputTokens 根据模型实际情况设置，模型列表展示能力信息
2. **Agent 面板**：上下文过大时自动触发压缩，而非直接报错

---

## 当前状态分析

### 问题 1：模型配置缺陷

- `modelGateway.listModels` 返回 `string[]`，丢弃了模型能力元信息
- `ModelProvider`/`ModelConfig` 类型无 `contextLength` 字段
- `model_providers` 表无存储模型能力的列
- `TaskConfigRow` 的 maxOutputTokens 是纯文本输入，默认硬编码 4096，无上限校验
- `ProviderFormModal` 模型下拉只显示模型名，不展示上下文窗口

### 问题 2：上下文过大处理缺陷

- `MODEL_CONTEXT_TOO_LONG` 标记为 `retryable: false`，立即返回错误
- `ModelService.callModel` 无降级策略，不重试、不压缩、不截断
- `AgentService.sendMessage` 收到错误后直接 `markRunFailed` 返回
- `AgentPanel` 仅显示"本次参考内容过长"，无引导回预览界面
- `ContextService.applyTokenLimit` 按类型预算整条排除，但无全局上限校验，必选项累加仍可能超限

## 改动方案

### 第一部分：模型配置优化

#### 1.1 新增类型 `ModelInfo`

**文件**：`src/types/index.ts`

```ts
/// 模型能力信息（来自 /v1/models 端点）
export interface ModelInfo {
  /// 模型 ID（如 'deepseek-chat'）
  id: string
  /// 上下文窗口大小（tokens），未知时为 null
  contextLength: number | null
  /// 最大输出 tokens，未知时为 null
  maxOutputTokens: number | null
}
```

#### 1.2 扩展 `modelGateway.listModels`

**文件**：`src/services/model/modelGateway.ts`

- 返回类型从 `Promise<string[]>` 改为 `Promise<ModelInfo[]>`
- 解析 `/v1/models` 响应时提取 `context_length`、`max_output_tokens` 等字段（不同服务商字段名可能不同，做兼容处理）
- 对常见模型（如 deepseek-chat=64K, deepseek-reasoner=64K, gpt-4o-mini=128K）提供内置 fallback 表，当 API 不返回时查表补充

#### 1.3 扩展 `ModelService.listProviderModels`

**文件**：`src/services/model/ModelService.ts`

- 返回类型从 `ServiceResult<string[]>` 改为 `ServiceResult<ModelInfo[]>`

#### 1.4 优化 `ProviderFormModal`

**文件**：`src/features/model/ModelSettingsPage.tsx`

- `modelOptions` 类型从 `string[]` 改为 `ModelInfo[]`
- 模型下拉项显示：`模型名 (上下文: 64K)`，能力未知时显示 `模型名 (上下文: 未知)`
- 选择模型后，将 `contextLength` 暂存到 provider 表单状态

#### 1.5 优化 `TaskConfigRow`

**文件**：`src/features/model/ModelSettingsPage.tsx`

- 选择服务商+模型后，自动拉取该模型的 `ModelInfo`
- maxOutputTokens 输入框改为带建议值的输入：
  - 显示模型实际上限提示：`最大输出 Token（模型上限: 8192）`
  - 输入值超过模型上限时校验失败：`不能超过模型最大输出 Token 上限 (8192)`
  - 模型上限未知时，仅校验正整数，并提示`未获取到模型上限，请谨慎设置`
- 新增"使用推荐值"快捷按钮，一键填入模型上限的 75%（预留 25% 给输入）

#### 1.6 扩展 `ModelProvider` 类型与表结构

**文件**：`src/types/index.ts` + `src-tauri/migrations/002_add_model_context_length.sql`（新建迁移）

- `ModelProvider` 新增 `defaultModelContextLength: number | null` 字段
- `model_providers` 表新增 `default_model_context_length INTEGER` 列
- `modelRepository.ts` 更新 `PROVIDER_FIELD_MAP` 和 `mapProvider`
- `ModelService.createProvider`/`updateProvider` 保存 contextLength

### 第二部分：上下文自动压缩

#### 2.1 新增 `ContextCompactor` 工具模块

**文件**：`src/services/context/ContextCompactor.ts`（新建）

职责：当上下文 token 超过模型上限时，按优先级自动压缩/裁剪。

```ts
/// 压缩结果
type CompactResult = {
  /// 压缩后的条目列表
  entries: ContextEntry[]
  /// 压缩后的总 token 估算
  totalTokens: number
  /// 被压缩/裁剪的条目信息（用于日志和用户提示）
  compactedItems: Array<{
    refId: string
    title: string
    action: 'truncated' | 'excluded'
    originalTokens: number
    newTokens: number
  }>
}

/// 压缩上下文以适配模型上限
export function compactContext(
  entries: ContextEntry[],
  maxTokens: number,
  options?: { preserveRequired?: boolean }
): CompactResult
```

压缩策略（按优先级执行）：
1. **必选项保护**：`required=true` 的条目不裁剪（但可截断文本）
2. **可选条目按优先级裁剪**：低优先级（priority 数字大）的可选条目先排除
3. **单条截断**：若仍超限，对非必选条目按 `truncateLen` 减半截断
4. **必选项截断**：最后手段，对必选项的 `preview` 文本截断（保留 `user_instruction` 和 `selected_text` 不动，截断 `project_rules`）
5. **全局上限**：确保最终 `totalTokens <= maxTokens`

#### 2.2 扩展 `ContextService.previewContext`

**文件**：`src/services/context/ContextService.ts`

- 新增可选参数 `modelMaxTokens?: number`（模型上下文窗口大小）
- 若传入 `modelMaxTokens`，调用 `compactContext` 进行预压缩
- 在 `ContextPreview` 中新增 `compactionInfo` 字段，记录压缩情况供 UI 展示

```ts
// ContextPreview 新增字段
export interface ContextPreview {
  // ... 现有字段
  /// 上下文压缩信息（若发生压缩）
  compactionInfo?: {
    originalTokens: number
    compactedTokens: number
    compactedItems: Array<{ title: string; action: string }>
  }
}
```

#### 2.3 修改 `AgentService.sendMessage` 自动降级

**文件**：`src/services/agent/AgentService.ts`

- 调用 `callModel` 前查询模型的 `contextLength`
- 将 `MODEL_CONTEXT_TOO_LONG` 错误标记为可重试（在 `errors.ts` 中改 `retryable: true`）
- 在 `sendMessage` 中捕获 `MODEL_CONTEXT_TOO_LONG` 错误后：
  1. 查询 ContextPack 的完整 entries
  2. 调用 `compactContext` 压缩到模型上限的 80%
  3. 用压缩后的 contextSummary 重新构造消息
  4. 重试调用 `callModel`（最多 1 次自动重试）
- 若自动压缩后仍失败，返回带 `suggestedAction` 的错误，引导用户手动排除

#### 2.4 修改 `errors.ts`

**文件**：`src/constants/errors.ts`

- `MODEL_CONTEXT_TOO_LONG` 改为 `retryable: true`
- 新增 `suggestedAction: '正在自动压缩上下文重试...'`
- 新增错误码 `MODEL_CONTEXT_COMPACT_FAILED`：自动压缩后仍超限

#### 2.5 优化 `AgentPanel` 错误处理

**文件**：`src/components/layout/AgentPanel.tsx`

- `handleConfirmContextPack` 中捕获 `MODEL_CONTEXT_TOO_LONG` 错误时：
  - 显示"上下文过大，正在自动压缩重试..."的 loading 状态
  - 自动压缩后重试发送
- 捕获 `MODEL_CONTEXT_COMPACT_FAILED` 错误时：
  - 显示"上下文过大，请返回排除部分内容"提示
  - 提供"返回上下文预览"按钮，重新打开 `ContextPreviewPanel`

#### 2.6 优化 `ContextPreviewPanel` 展示压缩信息

**文件**：`src/components/agent/ContextPreviewPanel.tsx`

- 当 `preview.compactionInfo` 存在时，显示压缩提示条：
  ```
  ℹ️ 已自动压缩 3 项参考内容（4500 → 3200 tokens）以适配模型上限
  ```
- Token 进度条的最大值改为模型实际上限（若已知）

## 假设与决策

### 决策 1：模型能力信息存储位置
**决策**：存储在 `model_providers.default_model_context_length`，而非每次调用时实时查询。
**理由**：避免每次 Agent 调用都额外请求 `/v1/models`；用户配置服务商时已拉取过模型列表，此时缓存能力信息即可。

### 决策 2：自动压缩触发时机
**决策**：双重触发——发送前预压缩 + 失败后降级重试。
**理由**：
- 预压缩：在 `previewContext` 阶段若已知模型上限，主动压缩，避免无谓的失败请求
- 降级重试：模型实际 token 计算（BPE）与估算（中文1:1/英文4:1）有偏差，预压缩可能仍超限，需失败后兜底

### 决策 3：压缩策略优先级
**决策**：排除低优先级可选条目 > 截断可选条目文本 > 截断必选规则条目 > 保护指令和选区。
**理由**：用户指令和选区是任务核心，不可动；项目规则可截断但不可完全移除；可选项按优先级牺牲。

### 决策 4：模型能力 fallback 表
**决策**：内置常见模型的能力表（deepseek-chat/reasoner 64K、gpt-4o 128K、gpt-4o-mini 128K、claude-3.5-sonnet 200K 等）。
**理由**：OpenAI 标准 `/v1/models` 不返回 context_length，但 DeepSeek、OpenRouter 等会返回；fallback 表保证主流模型能力可知。

### 决策 5：数据库迁移
**决策**：新建 `002_add_model_context_length.sql` 迁移，给 `model_providers` 加 `default_model_context_length` 列。
**理由**：遵循现有迁移规范，不修改 001 迁移文件。

## 验证步骤

1. **typecheck**：`npx tsc --noEmit` 无错误
2. **模型配置页面**：
   - 新建服务商 → 连接成功后模型下拉显示 `模型名 (上下文: 64K)`
   - 任务模型配置 → 选择模型后 maxOutputTokens 显示模型上限提示
   - 输入超过上限的值 → 校验失败
3. **上下文自动压缩**：
   - 构造超大上下文（如选区 5000 字 + 全项目资料）→ 发送
   - 预期：自动压缩后发送成功，或失败后显示引导提示
   - 检查 `agent_runs` 表的 `error_code` 字段记录压缩情况
4. **Agent 面板**：
   - 上下文过大时显示"正在自动压缩重试..."loading
   - 压缩失败时显示"返回上下文预览"按钮
