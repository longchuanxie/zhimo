本计划严格遵循 AGENTS.md：Service 分层、中文 UI 优先、Heroicons 图标、统一错误处理、不滥用 any、组件职责单一。

### 阶段 C-1：编辑器插入弹框收尾

#### C.7 新建 `src/components/editor/modals/FigureInsertModal.tsx`

**Props**：
```ts
type Props = {
  editor: Editor | null
  documentId: string
  projectId: string
  open: boolean
  kind: 'figure' | 'table'   // 由 editorModal 区分
  onClose: () => void
}
```

**职责**：
- 复用 Modal 组件，maxWidthClass="max-w-2xl"
- 共通字段：`caption`（必填，textarea）、`label`（可选，input，placeholder 如 `fig:einstein`）、`note`（可选，textarea）
- `kind === 'figure'` 分支：
  - 按钮"选择图片"→ 调用 `open` from `@tauri-apps/plugin-dialog`（filters: images）→ 读取为 base64（用 Tauri `readFile` 或 `fetch`+FileReader 兜底；MVP 用 FileReader 兜底读取 blob 转 base64）
  - 预览缩略图（`<img src={imageData}>`）
  - 字段写入 `imageData`（base64 字符串，无 `data:` 前缀），`imagePath` 留空
- `kind === 'table'` 分支：
  - 简易表格编辑器：行数/列数输入（默认 3×3）+ 单元格 `<textarea>` 网格
  - 提交时组装为 TipTap Table JSON 写入 `tableData`（结构：`{ type:'table', content:[{type:'tableRow', content:[{type:'tableCell',content:[{type:'paragraph'}]}]}] }`）
- 共通：`sourceId` 通过 `SourcePickerModal` 选择（`allowSkip=true`，材料真实性保障，但允许跳过以兼容自创图表）
- 提交流程：
  1. 校验 `caption` 非空
  2. 调用 `createFigure({ projectId, documentId, kind, caption, label, note, sourceId, imageData, tableData })`
  3. 成功后 `editor.chain().focus().insertFigureBlock({ figureId: result.data.id, kind, label }).run()`
  4. `toast.success('图表已插入')`，`onClose()`，重置表单
- 失败：`toast.error('插入失败：' + result.error.message)`
- `useEffect` 在 `open` 变化时重置表单状态

**关键约束**：
- 不直接访问数据库，仅通过 FigureService
- 中文 UI 文案
- sourceId 关联为材料真实性弱约束（允许跳过，但 UI 提示"建议关联来源资料以保障材料真实性"）

#### C.8 新建 `src/components/editor/modals/EquationInsertModal.tsx`

**Props**：
```ts
type Props = {
  editor: Editor | null
  documentId: string
  projectId: string
  open: boolean
  onClose: () => void
}
```

**职责**：
- 复用 Modal 组件，maxWidthClass="max-w-xl"
- 字段：`latex`（textarea，必填，placeholder 如 `E = mc^2`）、`label`（可选，placeholder 如 `eq:energy`）
- 实时预览：`useEffect` 监听 `latex` 变化，动态 `import('katex')` → `katex.renderToString(latex, { throwOnError: false, displayMode: true })` → 用 `dangerouslySetInnerHTML` 渲染到预览区
  - 注意：预览用 `throwOnError: false`（不抛错，显示红色错误标记），提交校验用 `validateLatex`（throwOnError: true）
- 提交流程：
  1. 校验 `latex` 非空
  2. 调用 `validateLatex(latex)` 校验语法（失败则 toast 提示，不提交）
  3. 调用 `createEquation({ projectId, documentId, latex, label })`
  4. 成功后 `editor.chain().focus().insertEquationBlock({ equationId: result.data.id, latex, label }).run()`
  5. `toast.success('公式已插入')`，`onClose()`，重置表单
- 失败：`toast.error('插入失败：' + result.error.message)`
- KaTeX 动态 import，避免阻塞首屏

## 四、假设与决策

### 假设（已通过探索核实）
1. `EditorToolbar` 已通过 `setEditorModal` 触发所有 6 类弹框，故只需在 `EditorModals` 容器中订阅 `editorModal` 即可联动
2. `FigureService.createFigure` 的 `imageData` 接受 base64 字符串，`tableData` 接受 `unknown`（TipTap JSON）
3. `EquationService.validateLatex` 已封装 KaTeX 动态 import 与 `throwOnError: true`，预览阶段直接用 `throwOnError: false` 渲染
4. `PaperService.getDocumentPaperMeta` 一次性返回统计 + issues，无需多次调用
5. `NAV_LABELS.references` 与 `NAV_ICONS.references` 已存在，无需新增常量

### 决策
1. **图表图片读取**：MVP 用浏览器 FileReader 读取 Tauri dialog 选择的文件为 base64（兼容性好，避免引入 Tauri readFile 二进制处理复杂度）。若文件过大提示警告（不阻断）
2. **表格编辑器**：MVP 用行列输入 + textarea 网格（不引入复杂表格组件），提交时组装为 TipTap Table JSON
3. **完整性面板位置**：放在编辑器下方（非右侧抽屉），避免与现有 AgentPanel 抢占右侧空间；可折叠
4. **sourceId 关联策略**：图表关联来源为"弱约束"（允许跳过，仅警告），与 PaperService.checkIntegrity 中 `missing_source` 为警告级一致；引文关联参考文献为"强约束"（必须选择，已在 CitationInsertModal 实现）
5. **KaTeX 预览**：用 `throwOnError: false` 渲染（显示红色错误不崩溃），提交时用 `validateLatex`（throwOnError: true）严格校验
6. **不新增技术债**：所有新代码严格类型化，不引入 any
