## 一、摘要

在 AI 原生写作项目系统中完成论文写作闭环。剩余工作分三阶段顺序推进:
- **阶段 A**:编辑器序列化器收尾(`editorToDocx` + `editorToPlainText` 改造)
- **阶段 B**:导出增强(LatexExporter + DocxExporter + ExportService/UI 改造 + 导出前真实性阻断)
- **阶段 C**:Agent 增强 + 参考文献管理 UI + 插入类 UI + 完整性面板 + 路由集成

**核心设计原则(论文写作严谨性)**:
- **材料真实性**:Source → BibliographicMetadata → Reference → Citation 全链路可追溯;Figure 强制建议关联 sourceId;导出前 `orphan_citation` 阻断;Agent 上下文只注入 `sourceStatus=active && aiUsageAllowed=true` 资料 + 未软删 reference。
- **格式严谨性**:GB/T 7714-2015 国标引文;章节编号在导出阶段按 heading 顺序生成 `1 / 1.1 / 1.1.1`;图表/公式独立编号序列;交叉引用 label 文档内唯一。
- **公式严谨性**:LaTeX 语法 KaTeX 校验(`throwOnError: true`);块级公式 `\begin{equation}\label{}`;行内公式 `$...$` 不入库。
- **图表严谨性**:caption 必填;label 文档内唯一;figure/table 自动编号独立序列;图片 base64 内联(MVP)。
- **可维护性**:Service 返回 `ServiceResult<T>`;常量集中;组件不直接访问 DB;单组件 ≤250 行;错误码集中。

### 阶段 A:编辑器序列化器收尾

#### A.1 `src/components/editor/editorToDocx.ts`(新建)

**职责**:TipTap JSON → `docx.Document` 对象,用于生成真实 `.docx` 二进制。

**依赖**:`docx` 库已安装(package.json)。

**关键签名**:
```ts
import type { JSONContent } from '@tiptap/react'
import {
  Document, Paragraph, TextRun, HeadingLevel, Table, TableRow, TableCell,
  WidthType, ImageRun, AlignmentType,
} from 'docx'
import type { Reference, Figure, Equation, Citation } from '@/types'

export type EditorToDocxOptions = {
  references: Reference[]
  figures: Figure[]
  equations: Equation[]
  citations: Citation[]
  /// 图片解码缓存(figureId → Buffer),由调用方预解码
  figureImageBuffers?: Map<string, Buffer>
}

export function editorToDocx(
  content: JSONContent | null,
  options: EditorToDocxOptions,
): Document
```

**转换规则**:
- `doc` → 顶层 `Document({ sections: [{ children: [...] }] })`
- `heading` level 1/2/3 → `Paragraph({ heading: HeadingLevel.HEADING_1/2/3 })`
- `paragraph` → `Paragraph({ children: TextRun[] })`
- `blockquote` → `Paragraph({ indent: { left: 720 } })`(0.5 英寸缩进)
- `bulletList`/`orderedList` → `Paragraph({ bullet: { level: 0 } })` / `numbering: { reference: 'numbered-list', level: 0 }`
- `codeBlock` → `Paragraph({ children: [TextRun({ text, font: 'Consolas' })] })`
- `horizontalRule` → `Paragraph({ border: { bottom: { color: '999999', size: 6, style: BorderStyle.SINGLE } } })`
- `figureBlock`(kind=figure)→ `Paragraph({ children: [ImageRun({ data, transformation: { width, height } })] })` + 题注 `Paragraph({ alignment: AlignmentType.CENTER, children: [TextRun({ text: '图 N: caption', italics: true })] })`;图片 Buffer 由 options.figureImageBuffers 提供,缺失时降级为 `[图片缺失]` 文本
- `figureBlock`(kind=table)→ `Table({ rows: [...] })` + 题注 Paragraph
- `equationBlock` → `Paragraph({ alignment: AlignmentType.CENTER, children: [TextRun({ text: latex, font: 'Cambria Math' })] })` + `(N)` 编号(TextRun 右对齐)— **TD-PAPER-002 降级方案**,公式以 LaTeX 源码呈现,前置注释 `【公式 LaTeX 源码,建议在 Word 中通过公式编辑器渲染】`
- `table` → `Table({ rows: [...] })`(遍历 tableRow/tableCell)
- `image` → `Paragraph({ children: [ImageRun({ data: base64Decode(src), transformation: { width: 600, height: 400 } })] })`(base64 解码,失败降级为文本)
- 行内 marks:
  - `bold` → `TextRun({ bold: true })`
  - `italic` → `TextRun({ italics: true })`
  - `code` → `TextRun({ font: 'Consolas' })`
  - `citation` → `TextRun({ text: citation.inlineText || '[?]' })`(从 citations 选项查)
  - `crossReference` → `TextRun({ text: label || '?' })`
  - `footnote` → `TextRun({ text: `[脚注: ${content}]` })`(简化,真正脚注需 docx.footnotes,MVP 降级)
  - `mathematics` → `TextRun({ text: `$${text}$`, font: 'Cambria Math' })`(行内公式降级)

**实现要点**:
- 文件长度控制 ≤ 250 行,辅助函数拆分(`convertBlock` / `convertInline` / `convertTable` / `decodeBase64ToBuffer`)
- `escapeLatex` 不适用,DOCX 无需转义
- 图片宽度统一 600px(8.33 英寸 ×72 DPI),高度按比例缩放或固定 400px(MVP 简化)
- 章节编号由 `LatexExporter`/`DocxExporter` 在外层包装时添加前缀(不在 editorToDocx 内做)

**章节编号处理**(导出器外层处理,非 editorToDocx 内):
- 在 `DocxExporter` 调用 `editorToDocx` 前,遍历文档 JSON 给所有 heading 节点 attrs 添加 `numberPrefix`(如 "1.1 "),`editorToDocx` 读取 `attrs.numberPrefix` 作为 heading 文本前缀
- 同理 `LatexExporter` 在调用 `editorToLatex` 前注入

### 阶段 B:导出增强

#### B.1 `src/services/export/LatexExporter.ts`(新建)

**职责**:生成完整 `.tex` 文件内容(preamble + body + bibliography)。

**关键签名**:
```ts
import type { Document, Reference, Figure, Equation, Citation, CitationStyle } from '@/types'

export type ExportLatexOptions = {
  /// 文档类(默认 article)
  documentClass?: 'article' | 'report' | 'book'
  /// 字号(pt,默认 12)
  fontSize?: number
  /// 引用格式(默认 gbt7714_2015)
  citationStyle?: CitationStyle
  /// 是否生成参考文献表(默认 true)
  includeBibliography?: boolean
  /// 是否生成图表目录(默认 false)
  includeListOfFigures?: boolean
}

/// 主入口:接收多个文档 + 各文档的元数据,返回完整 .tex 内容
export async function exportToLatex(
  documents: Document[],
  /// 每个文档对应的 figures/equations/citations/references
  metadata: Array<{
    figures: Figure[]
    equations: Equation[]
    citations: Citation[]
    references: Reference[]
    /// 参考文献表(已格式化的字符串数组)
    bibliography: string[]
  }>,
  options?: ExportLatexOptions,
): Promise<string>
```

**实现步骤**:
1. **生成 preamble**:
   ```
   \documentclass[12pt,a4paper]{article}
   \usepackage[UTF8]{ctex}              % 中文支持
   \usepackage{amsmath,amssymb,amsfonts} % 数学
   \usepackage{graphicx}                % 图片
   \usepackage{hyperref}                % 交叉引用/超链接
   \usepackage{booktabs}                % 表格三线表
   \usepackage{geometry}                % 页边距
   \geometry{top=2.5cm,bottom=2.5cm,left=3cm,right=3cm}
   \usepackage{caption}                 % 图表题注
   \usepackage[backend=biber,style=numeric,sorting=none]{biblatex}  % 引用(MVP 简化,直接 \bibitem)
   ```
2. **章节编号注入**:遍历每个文档 JSON,为 heading 节点 attrs 添加 `numberPrefix`(独立计数器:level1=1,2,3...;level2 在父级下 1.1,1.2;level3 = 1.1.1)。多文档连续编号(每个文档视为独立章节,使用 `\section` 顶层)。
3. **调用 `editorToLatex`**:对每个文档 JSON 调用,传入 references/figures/equations/citations。各文档 body 用 `\section{文档标题}` 分隔。
4. **生成参考文献表**:
   ```
   \begin{thebibliography}{99}
   \bibitem{citationKey1} 参考文献1格式化文本
   \bibitem{citationKey2} 参考文献2格式化文本
   \end{thebibliography}
   ```
   引用顺序按 citations 的 prosemirrorPos 升序去重。
5. **图表目录**(可选):`\listoffigures` + `\listoftables`
6. **拼接**:`preamble + \begin{document} + title + body + bibliography + \end{document}`

**实现要点**:
- 文件长度 ≤ 250 行
- 章节编号算法封装为 `injectSectionNumbers(doc: JSONContent): JSONContent` 纯函数
- 多文档时第一个文档的 title 作为 `\title`,后续文档作为 `\section`

#### B.2 `src/services/export/DocxExporter.ts`(新建)

**职责**:使用 `docx` 库生成真实 `.docx` 二进制。

**关键签名**:
```ts
import type { Document, Reference, Figure, Equation, Citation, CitationStyle, ExportOptions } from '@/types'

export type ExportDocxOptions = {
  /// 引用格式(默认 gbt7714_2015)
  citationStyle?: CitationStyle
  /// 是否含图表(默认 true,false 时跳过 figureBlock)
  includeFigures?: boolean
  /// 是否含目录(默认 false)
  includeTOC?: boolean
  /// 字体(默认 '宋体')
  fontFamily?: string
  /// 字号(pt,默认 12)
  fontSize?: number
  /// 行距(默认 1.5)
  lineHeight?: number
}

/// 主入口:接收多个文档 + 元数据,返回 docx 二进制
export async function exportToDocx(
  documents: Document[],
  metadata: Array<{
    figures: Figure[]
    equations: Equation[]
    citations: Citation[]
    references: Reference[]
    bibliography: string[]
  }>,
  options?: ExportDocxOptions,
): Promise<Uint8Array>
```

**实现步骤**:
1. **预解码图片 Buffer**:遍历所有文档的 figures,对 `figure.imageData`(base64)解码为 Buffer,存入 `Map<figureId, Buffer>`(供 `editorToDocx` 使用)
2. **章节编号注入**:与 LatexExporter 共用 `injectSectionNumbers` 函数(提到独立工具文件 `injectSectionNumbers.ts`)
3. **调用 `editorToDocx`**:对每个文档 JSON 调用,生成 `Document` 对象
4. **拼接多文档**:docx 库支持 `Document({ sections: [...] })`,每个文档作为独立 section,首 section 含 title page
5. **参考文献页**:最后追加一个 section,标题"参考文献",每条 `Paragraph({ text: bibitem })`
6. **目录**(可选):使用 docx 库的 `TableOfContents` 类
7. **生成二进制**:`Packer.toBuffer(document)` 返回 Buffer,转 `Uint8Array`

**实现要点**:
- 文件长度 ≤ 200 行(转换逻辑由 editorToDocx 承担)
- `decodeBase64ToBuffer` 使用 `atob` + `Uint8Array.from`
- 字体/字号/行距通过 docx 的 `styles` 配置注入 `Document` 顶层
- 引用格式化文本由 `metadata.bibliography` 提供(已由 PaperService.getBibliographyForExport 生成 GB/T 7714 格式)

#### B.3 `src/services/export/injectSectionNumbers.ts`(新建,共用工具)

**职责**:为 TipTap JSON 中的 heading 节点注入 `numberPrefix` 属性,实现章节自动编号。

**签名**:
```ts
import type { JSONContent } from '@tiptap/react'

/// 为 heading 节点注入 attrs.numberPrefix(如 '1.1.1 ')
/// 多文档场景:起始计数器由 startFrom 参数指定
export function injectSectionNumbers(
  doc: JSONContent | null,
  startFrom?: { l1: number; l2: number; l3: number },
): { doc: JSONContent | null; next: { l1: number; l2: number; l3: number } }
```

**算法**:
- 深度优先遍历 doc
- 遇到 `node.type === 'heading'`,读取 `node.attrs.level`
- 维护 `counters[3]`(level 1/2/3)
- 递增对应 level,下级 level 重置为 0
- 生成前缀字符串 `'1.'` / `'1.1.'` / `'1.1.1'` + 空格
- 写入 `node.attrs.numberPrefix`
- 返回更新后的 doc 和最终计数器(供下一文档续号)

## 五、假设与决策

### 5.1 关键决策(本计划新增)

- **章节编号**:导出阶段处理,通过 `injectSectionNumbers` 共用工具注入 `attrs.numberPrefix`,在 `editorToLatex`/`editorToDocx` 渲染时作为前缀
- **DOCX 公式**:降级为 LaTeX 源码文本(TD-PAPER-002),不实现 MathML→PNG
- **DOCX 脚注**:降级为 `[脚注: content]` 文本(不使用 docx.footnotes)
- **图片解码**:base64 → Buffer 使用 `atob` + `Uint8Array.from`,在 `DocxExporter` 预解码并传入 `editorToDocx`
- **EditorModals 挂载位置**:Editor.tsx 内部(避免 editor 实例传递)
- **PaperIntegrityPanel**:DocumentEditorPage 右侧可展开面板(非 Modal)
- **导出前真实性阻断**:仅 `orphan_citation` 阻断,其余 warning 不阻断
- **Agent 论文 prompt**:仅 `check_source` taskType 切换为论文校对 prompt,其余沿用通用 prompt
- **ReferencesPage "从资料导入"**:通过 SourcePickerModal 选择 source(简化,不做复杂筛选)

### 5.2 沿用 v3 决策

- **引用格式**:GB/T 7714-2015(国标),MVP 仅实现此一种
- **导出格式**:LaTeX (.tex) + Word (.docx 真实格式,使用 docx 库)
- **Footnote**:自研轻量 mark,内容存 attrs.content
- **图片存储**:MVP 用 base64 内联
- **完整性阻断**:仅 `orphan_citation` 阻断导出

### 5.3 假设

- `docx` 库的 `Packer.toBuffer` 返回 Buffer,可转 `Uint8Array` 用于 `writeBinary`
- `docx` 库的 `ImageRun` 接收 `{ data: Buffer | Uint8Array | ArrayBuffer | string, transformation: { width, height } }`
- `ContextPack` 类型有 `boundObjectType`/`boundObjectId` 或可通过 entries 推断(实施时确认)
- `@tauri-apps/plugin-dialog` `open` 已在 ExportWizardModal 中使用,可在 FigureInsertModal 复用
