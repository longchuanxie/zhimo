## 二、设计原则与关键决策

### 2.1 论文写作四项严格要求的落地策略

**① 内容格式**
- 章节编号:已由 `injectSectionNumbers` 共用工具统一注入,LaTeX/DOCX 序列化器消费。
- 引用格式:MVP 锁定 GB/T 7714-2015(已有 `Gbt7714Formatter`),导出向导可选 `citationStyle`。
- 导出向导(B.5):新增"导出选项"步骤,仅 latex/docx 显示,包含引用格式/包含图表/包含目录/字体/字号/行距/页边距。

**② 公式**
- 块级公式:编辑器内 KaTeX 实时预览(`EquationBlockView` 已实现);入库时 `validateLatex` 强制校验;`EquationInsertModal`(C.8)复用该校验,输入时实时预览。
- 行内公式:TipTap `Mathematics` 扩展已加载。
- label 唯一性:`EquationService.createEquation` 已校验;`checkIntegrity` 兜底。

**③ 图表插入**
- figure/table 独立编号序列(`FigureService` 已实现)。
- caption 必填(`FigureService.createFigure` 已校验);`checkIntegrity` 兜底。
- 材料真实性:figure 可关联 `sourceId`,`FigureInsertModal`(C.7)通过 `SourcePickerModal`(C.3)选择来源。
- 交叉引用:`CrossReference` mark 通过 label 关联,`CrossReferenceInsertModal`(C.9)列出文档内 figure/equation 供选择。

**④ 材料真实性(最严格)**
- 保障链:`Source` → `BibliographicMetadata` → `Reference`(`sourceId` 可溯源)→ `Citation`(`referenceId` 强关联)→ `PaperService.checkIntegrity` 检测 `orphan_citation`。
- **导出前阻断**:已由 ExportService.`checkOrphanCitationBlock` 实现(仅 latex/docx 阻断)。
- Agent 校对(C.1/C.2):新增论文校对 prompt,当 `taskType==='format_text'` 且 `boundObjectType==='document'` 时启用,检查引文规范性、材料可溯源性、图表题注、公式语法。
- `SourcePickerModal`(C.3):创建 Reference / Figure 时强制引导从已导入资料选择来源(可跳过,但跳过时 figure 标记 `missing_source` 警告)。
- `PaperIntegrityPanel`(C.12):文档侧实时显示完整性 issues,`orphan_citation` 标红,`missing_source`/`missing_caption`/`invalid_latex`/`label_duplicate` 标黄。

### 2.2 技术债记录(MVP 取舍,沿用上一轮)

| 编号 | 内容 | 原因 | 后续修复 |
|---|---|---|---|
| TD-PAPER-002 | DOCX 公式以 LaTeX 源码文本输出 | MathML→PNG 转换复杂 | 后续接入 `mathjax-node` |
| TD-PAPER-006 | DOCX 脚注以 `[脚注: content]` 文本输出 | `docx` 库 Footnote API 复杂 | 后续使用 `docx.Footnotes` |
| TD-PAPER-007 | DOCX 交叉引用以编号文本输出 | 无动态字段支持 | 后续使用 Word 域代码 |
| TD-PAPER-008 | Figure 图片 MVP 使用 base64 内联 | 不实现本地文件复制 | 后续支持本地图片资源管理 |

### 2.3 关键决策

1. **Agent prompt 切换不新增 taskType**:复用现有 `format_text`,通过 `boundObjectType==='document'` 判断启用论文 prompt,避免改动 `AgentTaskType` 枚举和模型配置映射。
2. **EditorModals 挂载位置**:挂载在 `Editor` 组件内(非全局),因为所有插入操作需要 `editor` 实例和 `documentId`/`projectId` 上下文。
3. **完整性面板刷新策略**:打开时立即查询 + 编辑器 update 时 debounce 2 秒重新查询,避免频繁查库。
4. **ReferencesPage 模式**:复用 `KnowledgeListPage` 的列表+创建+删除模式,但新增"从资料导入"按钮(调用 `ReferenceService.importFromSource`)。
5. **SourcePickerModal 复用**:同时供 `ReferenceFormModal`(关联参考文献来源)和 `FigureInsertModal`(关联图表来源)使用,Props 含 `allowSkip` 控制是否允许跳过。
6. **导出选项默认值**:与 `ExportService.DEFAULT_EXPORT_OPTIONS` 一致(宋体/12pt/1.5/2.54cm 边距),向导内本地 state 初始化即用此默认值。

**实现**:
```tsx
import { useAppStore } from '@/stores/appStore'
import { CitationInsertModal } from './CitationInsertModal'
import { FigureInsertModal } from './FigureInsertModal'
import { EquationInsertModal } from './EquationInsertModal'
import { CrossReferenceInsertModal } from './CrossReferenceInsertModal'
import { FootnoteInsertModal } from './FootnoteInsertModal'

export function EditorModals({ editor, documentId, projectId }: Props) {
  const editorModal = useAppStore((s) => s.editorModal)
  const setEditorModal = useAppStore((s) => s.setEditorModal)
  const close = () => setEditorModal(null)

  return (
    <>
      {editorModal === 'citation' && (
        <CitationInsertModal editor={editor} documentId={documentId} projectId={projectId} open onClose={close} />
      )}
      {editorModal === 'figure' && (
        <FigureInsertModal editor={editor} documentId={documentId} projectId={projectId} kind="figure" open onClose={close} />
      )}
      {editorModal === 'table' && (
        <FigureInsertModal editor={editor} documentId={documentId} projectId={projectId} kind="table" open onClose={close} />
      )}
      {editorModal === 'equation' && (
        <EquationInsertModal editor={editor} documentId={documentId} projectId={projectId} open onClose={close} />
      )}
      {editorModal === 'crossReference' && (
        <CrossReferenceInsertModal editor={editor} documentId={documentId} projectId={projectId} open onClose={close} />
      )}
      {editorModal === 'footnote' && (
        <FootnoteInsertModal editor={editor} open onClose={close} />
      )}
    </>
  )
}
```

## 五、假设与决策

1. **Agent prompt 切换不新增 taskType**:复用 `format_text` + `boundObjectType==='document'` 判断,避免改动枚举和模型配置。
2. **EditorModals 挂载在 Editor 内**:所有插入操作需要 editor 实例。
3. **完整性面板为右侧抽屉**:不占用编辑器空间,可随时关闭。
4. **ReferencesPage 复用 KnowledgeListPage 模式**:列表+创建+删除,新增"从资料导入"。
5. **SourcePickerModal 复用**:同时供 ReferenceFormModal 和 FigureInsertModal 使用。
6. **导出选项默认值**:与 `ExportService.DEFAULT_EXPORT_OPTIONS` 一致。
7. **ExportService 已有 `any[]` 技术债**:`mergeDocumentsForExport` 中 figures/equations/citations 使用 `any[]`,本轮替换为具体类型(`Figure[]`/`Equation[]`/`Citation[]`)以满足 lint。
