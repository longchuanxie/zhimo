> 本计划承接上一会话已完成的工作,聚焦剩余阶段。所有设计遵循 AGENTS.md 规范:Service 分层、类型明确、中文 UI 优先、材料真实性保障。

## 一、摘要

在 AI 原生写作项目系统中完成论文写作闭环,覆盖**内容格式、公式、图表插入、材料真实性**四项严格要求。剩余工作分四个阶段顺序推进:阶段3(Service 业务层收尾)→ 阶段2(编辑器扩展)→ 阶段4(导出增强)→ 阶段5(Agent + UI + 真实性校验)。

**核心设计原则**:
- **材料真实性**:Source → BibliographicMetadata → Reference → Citation 全链路可追溯;Figure 必须关联 sourceId(警告级);禁止 AI 输出直接覆盖正文。
- **格式严谨性**:GB/T 7714-2015 国标引文;LaTeX 公式经 KaTeX 校验;图表/公式自动编号;交叉引用解析。
- **可维护性**:Service 返回 `ServiceResult<T>`;常量集中;组件不直接访问 DB;单组件 ≤250 行。

## 三、提议的变更

### 阶段3:Service 业务层收尾

#### 3.1 `src/services/figure/FigureService.ts`(新建)

**职责**:图表 CRUD + 自动编号 + 悬空交叉引用检测。

**关键函数**:
- `listFiguresByDocumentId(documentId, kind?)` — 透传 figureRepository.listFiguresByDocument
- `getFigure(figureId)` — 查询,不存在返回 `NOT_FOUND`
- `createFigure(input)` — 校验 caption 非空;调用 `getMaxFigureNumber(documentId, kind) + 1` 分配编号(kind 独立序列);若提供 label 校验文档内不冲突(冲突返回 `FIGURE_NUMBER_CONFLICT`);若提供 sourceId 校验存在
- `updateFigure(input)` — 修改 label 时重新校验唯一性
- `deleteFigure(figureId)` — 软删除,返回被引用计数警告
- `renumberFigures(documentId)` — 按 prosemirrorPos 升序,figure 与 table 各自独立重新编号
- `checkOrphanCrossRefs(documentId)` — 检测被交叉引用但已删除的 figure(扫描 CrossReference mark 的 targetId,与现有 figure 比对)— 注:此函数依赖编辑器 JSON,先实现为"接收 figureId 列表 + 引用列表"的纯函数,UI 层调用时传入

**实现要点**:遵循 CardService 模式,返回 `ServiceResult<T>`,使用 `ok/err/fromUnknown`。

#### 3.2 `src/services/equation/EquationService.ts`(新建)

**职责**:公式 CRUD + 自动编号 + label 唯一性 + LaTeX 语法校验。

**关键函数**:
- `listEquationsByDocumentId(documentId)` — 透传
- `getEquation(equationId)` — 查询
- `createEquation(input)` — 校验 latex 非空;调用 `getMaxEquationNumber(documentId) + 1` 分配编号;若提供 label 调用 `findEquationByLabel` 校验文档内唯一(冲突返回 `EQUATION_LABEL_DUPLICATE`);调用 `validateLatex` 校验语法(失败返回 `LATEX_SYNTAX_INVALID`)
- `updateEquation(input)` — 修改 label 时重新校验唯一性;修改 latex 时重新校验语法
- `deleteEquation(equationId)` — 软删除
- `renumberEquations(documentId)` — 按 prosemirrorPos 升序重新编号
- `validateLatex(latex)` — **动态 import katex**,调用 `katex.renderToString(latex, { throwOnError: true })`,捕获异常返回 `ServiceResult<void>`;未安装 katex 时返回 ok(降级,不阻塞)

**实现要点**:
```ts
async function validateLatex(latex: string): Promise<ServiceResult<void>> {
  try {
    const katex = (await import('katex')).default
    katex.renderToString(latex, { throwOnError: true, displayMode: true })
    return ok(undefined)
  } catch (e) {
    return err({ code: LATEX_SYNTAX_INVALID, message: '公式 LaTeX 语法错误', retryable: false, detail: e instanceof Error ? e.message : String(e) })
  }
}
```

#### 3.3 `src/services/paper/PaperService.ts`(新建,协调层)

**职责**:聚合文档论文元数据 + 完整性检查 + 真实性校验。不直接访问 DB,调用 CitationService/FigureService/EquationService/ReferenceService。

**关键函数**:
- `getDocumentPaperMeta(documentId)` — 聚合 citationCount/figureCount/tableCount/equationCount/wordCount + issues 列表,返回 `DocumentPaperMeta`
- `checkIntegrity(documentId)` — 聚合完整性检查,返回 `IntegrityIssue[]`:
  - 调用 `CitationService.checkOrphanCitations` → `orphan_citation`
  - 调用 `FigureService.checkOrphanCrossRefs` → `orphan_cross_ref`
  - 扫描 figure 缺 caption → `missing_caption`(理论上 createFigure 已校验,这里兜底)
  - 扫描 figure 缺 sourceId → `missing_source`(警告级)
  - 扫描 equation label 重复 → `label_duplicate`
  - 扫描 equation latex 无效 → `invalid_latex`
- `getBibliographyForExport(documentId, style)` — 透传 `CitationService.generateBibliography`(供导出层调用)
- `getFiguresForExport(documentId)` — 返回 figure 列表(供导出层嵌入图片)
- `getEquationsForExport(documentId)` — 返回 equation 列表(供导出层生成 LaTeX 公式)

**实现要点**:作为协调层,不重复业务校验,只做聚合。

#### 3.7 序列化器(2 个新建 + 1 个改造)

##### 3.7.1 `src/components/editor/editorToLatex.ts`(新建)

**职责**:将 TipTap JSON 序列化为 LaTeX 源码。

**关键函数**:`editorToLatex(doc: TipTapDoc, options: { references: Reference[]; figures: Figure[]; equations: Equation[]; citations: Citation[] }): string`

**转换规则**:
- heading → `\section{}` / `\subsection{}` / `\subsubsection{}`
- paragraph → 段落
- blockquote → `\begin{quote}...\end{quote}`
- bulletList/orderedList → `\begin{itemize}` / `\begin{enumerate}`
- figureBlock → `\begin{figure}\includegraphics{}\caption{}\end{figure}`
- table → `\begin{table}\begin{tabular}...\end{tabular}\end{table}`
- equationBlock → `\begin{equation}\label{}latex\end{equation}`
- citation mark → `\cite{}`(numeric)或 `\citeauthor{} \citeyear{}`(author_year)
- crossReference mark → `\ref{}`
- footnote mark → `\footnote{}`
- 数学行内(inline) → `$...$`

**输出**:完整 `.tex` 文档 preamble + body + bibliography(`\bibitem` 列表)。

##### 3.7.2 `src/components/editor/editorToDocx.ts`(新建)

**职责**:将 TipTap JSON + 图表数据转换为 `docx` 库的 Document 对象,用于生成真实 `.docx`。

**关键函数**:`editorToDocx(doc: TipTapDoc, options): docx.Document`

**转换规则**:使用 `docx` 库的 Paragraph/TextRun/Table/ImageRun/HeadingLevel,公式渲染为 PNG(MathML 转图片,MVP 可降级为文本)或 KaTeX 生成的 SVG。图片通过 `ImageData` base64 解码为 Buffer。

**变更点**:增加对新节点(figureBlock/equationBlock/citation mark 等)的处理,生成可读纯文本(如 `[图表 1: xxx]` / `[公式 1]` / `[1]`)。

## 五、假设与决策

### 5.1 关键决策(已确认,本计划沿用)
- **引用格式**:GB/T 7714-2015(国标),MVP 仅实现此一种
- **导出格式**:LaTeX (.tex) + Word (.docx 真实格式,使用 docx 库)
- **CHECK 约束**:006 迁移重建表(已完成)
- **实施范围**:全量阶段3→2→4→5

### 5.2 本计划新增决策
- **SectionNumber**:降级为导出阶段处理,不创建独立扩展(MVP 简化)
- **Footnote**:自研轻量 mark,内容存 attrs.content,不实现独立编辑器 UI
- **公式渲染**:编辑器内用 KaTeX 渲染 SVG/HTML;DOCX 导出降级为文本(带 LaTeX 源码注释),不实现 MathML→PNG
- **图片存储**:MVP 用 base64 内联(imageData),大图后续支持 imagePath 本地文件
- **完整性阻断**:仅 `orphan_citation` 阻断导出,其余为警告

### 5.3 假设
- `@tiptap/extension-mathematics` 的 regex 可配置为 `$...$` 行内公式
- `docx` 库支持 ImageRun 接收 base64 解码后的 Buffer
- 路由配置文件位于 `src/app/` 或 `src/routes/`(阶段5 实施时定位)
