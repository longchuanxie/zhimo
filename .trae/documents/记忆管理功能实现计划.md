# 记忆管理功能实现计划

## 概述

在项目设置页面中新增"Agent 记忆管理"区块，支持记忆的增删改查。记忆是项目级的跨会话上下文，用户可手动添加、编辑、删除记忆条目，也可查看自动提取的对话摘要。

## 当前状态分析

### 已有基础设施

- **Service 层完整**：`AgentMemoryService.ts` 已提供 7 个 API（`listMemories`、`createMemory`、`updateMemory`、`removeMemory`、`removeMemoriesByProject`、`getMemory`、`recallMemories`），返回 `ServiceResult<T>` 统一结构。
- **类型定义完整**：`AgentMemory`（id/projectId/sourceThreadId/kind/content/confidence/createdAt/updatedAt）、`AgentMemoryKind`（5 种：preference/fact/decision/style/summary）。
- **数据库表已建**：`agent_memories` 表（migration 004）。
- **UI 基础组件齐全**：`Modal`、`ConfirmDialog`、`AlertDialog`、`EmptyState`、`ErrorState`、`LoadingState`、`AppIcon`。
- **异步状态管理**：`useAsync` Hook 支持 loading/error/success 三态 + `refresh()`。
- **项目设置页面**：`ProjectSettingsPage.tsx`（375 行），单页表单结构，使用 `FormField` 子组件。

### 缺口

- `listMemories`、`updateMemory`、`removeMemory`、`removeMemoriesByProject` 这 4 个 API 无任何调用方（仅 `createMemory` 和 `recallMemories` 被使用）。
- 无记忆管理 UI。
- `AgentMemoryKind` 无中文标签映射（散落在 `ContextService.ts` 的 `loadAgentMemories` 函数内部）。

## 实现方案

### 决策：嵌入项目设置页面

记忆是项目级的，放在项目设置页面中最自然。为避免 `ProjectSettingsPage` 超过 250 行阈值，将记忆管理拆分为独立组件 `MemoryManagementSection`，作为 section 嵌入。

### 文件变更清单

| 文件 | 操作 | 说明 |
|---|---|---|
| `src/constants/status.ts` | 修改 | 新增 `AGENT_MEMORY_KIND_LABEL` 中文映射 |
| `src/features/agent/MemoryManagementSection.tsx` | 新建 | 记忆管理区块组件（列表 + 新建/编辑弹窗 + 删除确认） |
| `src/features/project/ProjectSettingsPage.tsx` | 修改 | 在表单底部引入 `MemoryManagementSection` |

### 详细设计

#### 1. `src/constants/status.ts` — 新增记忆类型中文标签

在文件末尾新增：

```ts
import type { AgentMemoryKind } from '@/types'

export const AGENT_MEMORY_KIND_LABEL: Record<AgentMemoryKind, string> = {
  preference: '用户偏好',
  fact: '事实信息',
  decision: '关键决策',
  style: '写作风格',
  summary: '对话摘要',
}
```

同时将 `ContextService.ts` 中 `loadAgentMemories` 内部的局部 `kindLabels` 替换为引用此常量（消除重复）。

#### 2. `src/features/agent/MemoryManagementSection.tsx` — 新建组件

**组件签名**：

```ts
type MemoryManagementSectionProps = {
  projectId: EntityId
}
```

**内部结构**：

```
MemoryManagementSection
├── section 标题区（"Agent 记忆" + 说明文案 + "添加记忆"按钮 + "清空全部"按钮）
├── useAsync 加载记忆列表
├── 三态渲染
│   ├── loading → LoadingState
│   ├── error → ErrorState（含 retry）
│   └── success
│       ├── 空列表 → EmptyState（含"添加记忆"主操作）
│       └── 非空 → 记忆卡片列表
│           └── MemoryCard（子组件）
│               ├── 类型标签（StatusTag 样式）
│               ├── 内容预览（line-clamp-3）
│               ├── 置信度进度条
│               ├── 来源标识（自动提取/手动添加）
│               ├── 更新时间
│               └── 操作按钮（编辑、删除）
├── MemoryFormModal（子组件，新建/编辑共用）
│   ├── 类型选择（select：5 种 AgentMemoryKind）
│   ├── 内容输入（textarea，必填）
│   ├── 置信度滑块（range 0-1，步进 0.1）
│   └── 提交/取消
├── ConfirmDialog（删除单条确认）
└── ConfirmDialog（清空全部确认）
```

**关键实现细节**：

- **数据加载**：`useAsync(() => listMemories(projectId), [projectId])`
- **新建记忆**：`createMemory({ projectId, kind, content, confidence })`，`sourceThreadId` 为 null（手动添加）
- **编辑记忆**：`updateMemory(id, { content, confidence })`，注意 `kind` 不可编辑（Repository 层 `updateMemoryContent` 仅更新 content 和 confidence）
- **删除单条**：`removeMemory(id)` + `ConfirmDialog` 确认
- **清空全部**：`removeMemoriesByProject(projectId)` + `ConfirmDialog` 确认（danger 样式）
- **操作反馈**：`msg` state + 3 秒自动消失（与 ProjectSettingsPage 一致）
- **按钮 loading 态**：`submitting` / `deletingId` state 控制
- **编辑弹窗预填**：点击编辑时将当前记忆的 `content` 和 `confidence` 填入表单

**组件拆分**（同文件内子组件，遵循 CardListPage 模式）：

- `MemoryCard` — 单条记忆卡片展示
- `MemoryFormModal` — 新建/编辑表单弹窗

**预计行数**：约 200-240 行（主组件 + 2 个子组件），不超过 250 行阈值。

#### 3. `src/features/project/ProjectSettingsPage.tsx` — 引入记忆管理

在表单底部、操作按钮之前插入：

```tsx
{/* Agent 记忆管理 */}
<MemoryManagementSection projectId={projectId!} />
```

**改动量**：仅新增 1 行 import + 1 行 JSX，不改变现有表单逻辑。

**放置位置**：在"禁止规则"字段之后、"消息提示"之前，作为独立的 section。

### 样式规范

- section 标题：`text-base font-semibold text-ink`（与现有 FormField label 区分）
- 记忆卡片：`card p-4 space-y-2`（与 CardItem 一致）
- 类型标签：`tag-default` / `tag-brand` 等（与 StatusTag 一致）
- 置信度条：`h-1.5 rounded-full bg-line` + `bg-brand` 填充
- 操作按钮：`btn-ghost` + 图标（与 CardListPage 删除按钮一致）

### 图标选择（Heroicons 24px outline）

- 添加记忆：`PlusIcon`
- 编辑：`PencilIcon`
- 删除：`TrashIcon`
- 清空全部：`TrashIcon`（danger 样式）
- 空状态：`SparklesIcon`（与 ContextPreviewPanel 的 agent_memory 图标一致）
- 来源标识-自动提取：`Cog6ToothIcon`
- 来源标识-手动添加：`HandRaisedIcon`

## 假设与决策

1. **放置位置**：嵌入项目设置页面而非全局设置，因为记忆是项目级的。
2. **kind 不可编辑**：Repository 层 `updateMemoryContent` 仅更新 content 和 confidence，不更新 kind。这是合理的设计——记忆类型在创建时确定，后续只调整内容和置信度。
3. **置信度可编辑**：用户可调整置信度来控制记忆是否被召回（recallMemories 默认阈值 0.5）。
4. **不新增路由**：记忆管理作为项目设置的 section，不需要独立路由。
5. **sourceThreadId 不可编辑**：手动创建的记忆 sourceThreadId 为 null，自动提取的保留原值。
6. **清空全部**：提供批量删除功能，但需二次确认。
