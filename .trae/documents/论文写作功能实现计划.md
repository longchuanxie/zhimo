### 1.3 严格性约束落地点
- **材料真实性**:所有引文必须关联 `reference_id`,reference 必须关联 `source_id`(本地导入资料)或具备完整 `bibliographic_metadata`(DOI/ISBN/作者/年份/标题);Agent 系统提示强制"引用必须来自 reference 库,禁止编造";新增引用检查 Agent 扫描文档验证。
- **公式严谨性**:LaTeX 语法标准,块级公式自动编号(可手动指定 label),交叉引用解析后显示真实编号;导出 LaTeX 原生,Word 转 OMML。
- **图表可溯**:figure/table 必填 caption,可选 source_id 关联资料来源,编号自动生成,交叉引用不悬空。
- **格式规范**:GB/T 7714-2015 严格按国标格式化(专著 [M]/期刊 [J]/学位论文 [D]/会议 [C]/电子文献 [EB/OL] 等),参考文献表按引用顺序排序。

```sql
-- 1. 扩展 documents 表:论文格式配置
ALTER TABLE documents ADD COLUMN citation_style TEXT DEFAULT 'gbt7714_2015';
-- citation_style: 'gbt7714_2015' | 'apa7' | 'ieee' | 'mla9'(预留,MVP 仅实现 gbt7714_2015)

-- 2. 扩展 sources 表:书目元数据(材料真实性基础)
ALTER TABLE sources ADD COLUMN bibliographic_metadata TEXT;
-- JSON 结构: { authors: [{name, affiliation}], year, title, container, volume, issue, pages, doi, isbn, url, publisher, city, entryType: 'journal'|'book'|'conference'|'thesis'|'web'|'other' }

-- 3. 扩展 export_tasks 表:支持新格式(放宽 CHECK 约束)
-- SQLite 不支持 ALTER CHECK,通过新表迁移实现;MVP 简化为新增列兼容

-- JSON: { template, citationStyle, includeFigures, includeTOC, fontFamily, fontSize, lineHeight, margin }

-- 4. 参考文献库(项目级,一个 reference 可被多个文档引用)
CREATE TABLE IF NOT EXISTS references (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  source_id TEXT,                           -- 关联本地导入的资料(可空,允许手动录入)
  citation_key TEXT NOT NULL,               -- BibTeX 风格 key,项目内唯一,如 'smith2020ai'
  entry_type TEXT NOT NULL,                 -- 'journal'|'book'|'conference'|'thesis'|'web'|'other'
  title TEXT NOT NULL,
  authors_json TEXT NOT NULL,               -- [{name, affiliation?}]
  year INTEGER,
  container TEXT,                           -- 期刊名/书名/会议名
  volume TEXT,
  issue TEXT,
  pages TEXT,
  publisher TEXT,
  city TEXT,
  doi TEXT,
  isbn TEXT,
  url TEXT,
  access_date TEXT,                         -- 电子文献访问日期
  raw_metadata TEXT,                        -- 原始书目 JSON(导入时保留)
  is_deleted INTEGER NOT NULL DEFAULT 0,
  deleted_at TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  FOREIGN KEY (source_id) REFERENCES sources(id),
  UNIQUE(project_id, citation_key)
);
CREATE INDEX IF NOT EXISTS idx_references_project ON references(project_id, is_deleted, updated_at);
CREATE INDEX IF NOT EXISTS idx_references_source ON references(source_id);

-- 5. 引文实例(文档内每一次引用行为,关联 reference + 文档位置)
CREATE TABLE IF NOT EXISTS citations (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  document_id TEXT NOT NULL,
  reference_id TEXT NOT NULL,
  citation_format TEXT NOT NULL DEFAULT 'numeric', -- 'numeric'|'author_year'
  locator TEXT,                              -- 页码/章节定位,如 'p.123' 或 'ch.4'
  prefix TEXT,                               -- 引文前缀,如 '见'
  suffix TEXT,                               -- 引文后缀,如 '第2版'
  inline_text TEXT,                          -- 解析后的行内显示文本,如 '[1]' 或 '(Smith, 2020)'
  prosemirror_pos INTEGER,                   -- TipTap 文档位置(便于回溯定位)
  is_deleted INTEGER NOT NULL DEFAULT 0,
  deleted_at TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE,
  FOREIGN KEY (reference_id) REFERENCES references(id) ON DELETE CASCADE
);
CREATE INDEX IF NOT EXISTS idx_citations_document ON citations(document_id, is_deleted);
CREATE INDEX IF NOT EXISTS idx_citations_reference ON citations(reference_id);

-- 6. 图表(figure + table 统一管理,区分 kind)
CREATE TABLE IF NOT EXISTS figures (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  document_id TEXT NOT NULL,
  kind TEXT NOT NULL CHECK (kind IN ('figure','table')),  -- figure=图片,table=表格
  number INTEGER,                           -- 自动编号(figure 1,2,3...;table 独立序列)
  label TEXT,                               -- 用户可指定 label,如 'fig:architecture'
  caption TEXT NOT NULL,                    -- 题注(必填)
  note TEXT,                                -- 注释(可选)
  source_id TEXT,                           -- 来源资料(材料真实性,可空)
  image_path TEXT,                          -- 图片本地路径(figure)
  image_data TEXT,                          -- base64 内联(MVP 简化,小图)
  table_data TEXT,                          -- 表格 TipTap JSON(table)
  prosemirror_pos INTEGER,
  is_deleted INTEGER NOT NULL DEFAULT 0,
  deleted_at TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE,
  FOREIGN KEY (source_id) REFERENCES sources(id)
);
CREATE INDEX IF NOT EXISTS idx_figures_document ON figures(document_id, kind, is_deleted);
CREATE INDEX IF NOT EXISTS idx_figures_project ON figures(project_id, kind);

-- 7. 公式(块级公式编号管理;行内公式不入库,直接在 TipTap JSON 中)
CREATE TABLE IF NOT EXISTS equations (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  document_id TEXT NOT NULL,
  number INTEGER,                           -- 自动编号(公式 1,2,3...)
  label TEXT,                               -- 用户指定 label,如 'eq:euler'
  latex TEXT NOT NULL,                      -- LaTeX 源码
  prosemirror_pos INTEGER,
  is_deleted INTEGER NOT NULL DEFAULT 0,
  deleted_at TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE
);
CREATE INDEX IF NOT EXISTS idx_equations_document ON equations(document_id, is_deleted);
```

### 4.3 自定义扩展实现(新建 `src/components/editor/extensions/`)

#### 4.3.1 `CitationMark.ts` — 行内引文标注
- **类型**:Mark(行内标记)
- **属性**:`citationId`(关联 citations 表)、`referenceId`、`displayText`('[1]' / '(Smith, 2020)')
- **渲染**:`<sup class="citation" data-citation-id="...">[1]</sup>`
- **交互**:点击弹出浮层显示完整参考文献信息 + 跳转编辑
- **约束**:不可悬空(citationId 必须对应数据库记录)

#### 4.3.2 `FigureBlock.ts` — 图表块(figure/table 容器)
- **类型**:Node(block)
- **属性**:`figureId`(关联 figures 表)、`kind`('figure'|'table')、`number`、`label`、`caption`、`sourceId`
- **结构**:`<figure data-figure-id><img/></figure>` 或 `<figure data-figure-id><table/></figure><figcaption>`
- **编辑**:caption 可编辑;图片通过工具栏插入或拖拽;table 通过 Table 扩展
- **编号**:Service 层统一管理,编辑器只读显示

#### 4.3.3 `EquationBlock.ts` — 块级公式
- **类型**:Node(block)
- **属性**:`equationId`、`number`、`label`、`latex`
- **渲染**:KaTeX 渲染 + 右侧自动编号 `(1)`
- **编辑**:双击进入 LaTeX 源码编辑模式
- **行内公式**:由 Mathematics 扩展处理 `$...$`,不入库

#### 4.3.4 `CrossReference.ts` — 交叉引用
- **类型**:Mark(行内)
- **属性**:`targetType`('figure'|'table'|'equation'|'section')、`targetId`、`displayText`
- **渲染**:`<a class="cross-ref" data-target>图 1</a>`
- **交互**:点击跳转到目标位置;编号变化时由 Service 层批量更新 displayText
- **约束**:目标不存在时红色标记(悬空检测)

#### 4.3.5 `SectionNumber.ts` — 章节自动编号
- 基于 heading 扩展,新增 `sectionNumber` 属性
- 渲染时自动前置编号(1 / 1.1 / 1.1.1)
- 由 PaperService 在保存时统一计算

### 5.1 新增 Service 文件(遵循 §3.4 Service 可维护性要求)

#### 5.1.1 `src/services/citation/ReferenceService.ts` — 参考文献库管理

## 八、假设与决策

### 8.1 关键决策
1. **公式扩展选型**:优先 `@tiptap/extension-mathematics`(3.x 内置 KaTeX);若不可用,自研 NodeView。决策依据:避免 Pro 订阅依赖。
2. **参考文献存储**:独立 `references` 表而非复用 sources,因 reference 可手动录入(无本地文件)且需 citationKey 唯一约束。
3. **引文编号策略**:numeric 格式按文档出现顺序编号,Service 层统一管理(非编辑器实时),保存时 renumber。
4. **Word 公式方案**:MVP 优先 KaTeX → MathML → OMML 转换;复杂公式降级为图片。决策依据:`docx` npm 包原生 Math API 复杂,转换方案覆盖 90% 场景。
5. **LaTeX 导出范围**:生成可编译的 `.tex`(ctex 中文包),参考文献内联(`\begin{thebibliography}`),不依赖外部 .bib 文件。
6. **图表图片存储**:MVP 用 base64 内联 + 可选本地路径;小图内联避免文件管理复杂度,大图走 fileGateway。
7. **不引入 Zotero/外部文献管理集成**:MVP 范围内,通过 BibTeX 导入满足批量需求。

### 8.2 假设
- TipTap 3.x 的 `@tiptap/extension-mathematics` 在开源包中可用(需验证,否则自研)
- KaTeX 能覆盖论文常用公式(数学/物理/化学/统计),不依赖 MathJax 的高级扩展
- `docx` npm 包生成的 .docx 能被 WPS/Word 正常打开
- Tauri fileGateway 可扩展支持二进制写入(已有 `writeText`,新增 `writeBinary`)
- 用户本地有 LaTeX 环境(导出 .tex 由用户编译)或仅查看源码

### 8.3 非目标(MVP 不做)
- PDF 直接导出(依赖 LaTeX 或 Puppeteer,复杂度高,后置)
- Zotero/Mendeley 集成
- 多人协作引用管理
- 论文模板市场(仅提供默认通用学术论文模板)
- OCR 恢复(现有技术债,不在此范围)
